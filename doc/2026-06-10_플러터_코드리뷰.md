## 2. P1 — 앱이 MSA dev 토폴로지에 연결 불가

앱의 모든 API는 단일 base URL을 사용한다.

| 설정 | 값 | 현실 (MSA 이후) |
|---|---|---|
| dev baseUrl | `http://{host}:8080/api/v1` (`lib/core/config/app_config.dart:103`) | **8080에 듣는 서비스 없음.** compose/k8s 노출 포트는 8081(user)·8082(bible)·8083(note)·8084(ai)·8090(admin)뿐. k8s에 ingress/gateway 없음 |
| dev ttsBaseUrl | `http://{host}:8090` (`app_config.dart:119`) | 8090은 MSA에서 **admin-server가 점유** → TTS 요청이 admin-server로 가서 404. tts-service는 저장소에 코드 없음(빈 미추적 폴더) |
| `run-dev-web.ps1:50-51` | "서버 포트 8080 대기" | 모놀리식(8080) 잔재 — 현 토폴로지에선 영원히 대기 |

#465로 모놀리식 root 소스가 제거된 시점부터 앱-백엔드 연결 경로가 사라졌다. CI는 analyze/test만 수행하므로 잡히지 않는다.

해결 선택지 (Lead 결정 필요):

1. **(권장) compose에 nginx 등 리버스 프록시 추가** — 8080에서 경로 prefix별 분배(`/api/v1/auth|members|notifications|missions → 8081`, `/api/v1/bible|qt|study|music|praise → 8082`, `/api/v1/notes|sharing|reports → 8083`, `/api/v1/ai → 8084`). 앱 무수정, prod(api.qtai.com=LB 전제) 구조와 일관.
2. 앱에서 서비스별 base URL 분기 — AppConfig를 기능별 4개 URL로 확장. 앱 수정 범위 큼.
3. 임시: `--dart-define=DEV_BASE_URL`로 단일 서비스만 지정 — 기능 교차 시 한계.
