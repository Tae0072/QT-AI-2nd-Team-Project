# admin-web 코드 리뷰 (2026-06-10)

- 대상: `dev` @ `b534e8b6` · React+Vite+TS+antd, src 약 50파일
- 검증: Windows `npm ci && npm run build` 성공(4.22s) · 발견사항 원본 파일(파일:라인) 재확인 · 서버 컨트롤러 매핑과 직접 대조
- 리뷰 단계: 1/3 서버 → 2/3 flutter-app → **3/3 admin-web(본 문서)**

## 1. 종합

작은 코드베이스지만 구조(Context/API/라우팅 분리), strict TS, 친절한 주석, 백엔드 미준비 화면의 "준비 중" 안내까지 잘 정돈돼 있다. 핵심 문제는 flutter와 같은 뿌리의 **로그인 라우팅 단절 1건**.

## 2. P1 — 카카오 로그인 요청이 잘못된 서버로 간다 (실로그인 불가)

| 사실 | 근거 |
|---|---|
| 모든 API가 단일 axios 인스턴스 사용, baseURL 기본 `/api/v1`(상대경로) | `src/config/env.ts:5`, `src/api/client.ts:22` |
| dev에서 `/api`는 vite 프록시가 통째로 `VITE_API_PROXY_TARGET ?? http://localhost:8090`(admin-server)으로 전달 | `vite.config.ts:14,20-25` ("2026-06-10 결정 ③: admin-web은 처음부터 admin-server(8090) 기준") |
| 그런데 로그인 엔드포인트 `POST /api/v1/admin/auth/kakao`는 **service-user(8081)에만 존재** | service-user `AdminAuthController:23,30` (`/api/v1/admin/auth` + `/kakao`) |
| admin-server(8090)에는 해당 컨트롤러가 없고 `/api/v1/admin/**`은 `hasRole(ADMIN)` | admin-server 컨트롤러 전수 + `SecurityConfig:87` |

결과: 로그인 POST가 8090에 도착 → 컨트롤러 없음 + 인증 요구 → **401**. 카카오 JS 키를 넣어도 실로그인이 안 되는 구조다. `LoginPage.tsx:21` 주석은 "백엔드 준비 전엔 실행 불가"라 적었지만, 백엔드(service-user)는 이미 구현 완료 — 실제 원인은 **라우팅 경로 부재**.

현재는 dev 전용 "개발용 토큰" 입력(`LoginPage.tsx:111`, `import.meta.env.DEV` 가드 — prod 빌드 제거 확인)으로 우회 중.

수정(택1):

1. **(권장, 한 줄)** vite proxy에 구체 룰 추가 — `'/api/v1/admin/auth': { target: 'http://localhost:8081', changeOrigin: true }`를 `'/api'` 룰보다 위에. 게이트웨이 도입 전까지의 정합 패치.
2. flutter P1과 묶어 nginx 리버스 프록시 하나로 앱(8080)·관리자 라우팅 동시 해결 — 운영 구조와도 일관.

## 3. P2

1. **refreshToken 미사용**: 로그인 응답으로 받지만(`src/api/adminAuth.ts:13`) 갱신 로직이 없음 — 401이면 `clearToken()` 후 재로그인 유도(`client.ts:44-46`). access 30분 정책상 **30분마다 재로그인**. refresh 흐름 추가 권장. (`AuthContext`의 `refreshAdminInfo`는 관리자 정보 재조회일 뿐 토큰 갱신 아님)
2. **localStorage 토큰 저장** — XSS 시 토큰 탈취 가능. 의도적 임시 설계(주석 존재)이나 운영 전 HttpOnly 쿠키 등 재검토.
3. **임시 DTO**: AD-01/02/06 응답 타입이 `[key: string]: unknown` 임시 형태 — 백엔드 계약 확정 시 즉시 구체화.

## 4. P3

- 찬양 곡 숨김 API 함수 정의만 있고 UI 버튼 미연결.
- 카카오 SDK 로드 실패 시 사용자 안내 빈약(에러 메시지 위주).
- 빌드 산출물 단일 청크 1.15MB(antd) — code-split 여지.

## 6. 잘된 점 (확인 완료)

- 단일 axios + 요청 인터셉터(토큰 자동 부착) + 응답 인터셉터(401 토큰 정리, 에러 정규화) + `unwrap` 봉투 처리 — 일관된 API 레이어.
- 카카오 JS 키는 `VITE_KAKAO_JS_KEY` env 처리, 하드코딩 0건. 개발용 토큰 로그인은 DEV 가드로 prod 제거.
- tsconfig strict, `any`/`@ts-ignore` 없음. 입문자 배려 주석 우수.
- 계약 합의 기록(2026-06-10 admin-kakao-auth-contract)이 코드 주석에 남아 추적 가능.
