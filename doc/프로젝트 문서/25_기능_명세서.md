## 1. 문서 목적과 경계

이 문서는 `07_요구사항_정의서.md`의 F-ID를 구현 가능한 기능 단위로 풀어쓴다. 요구사항 포함 여부를 바꾸지 않는다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 상세 동작 | 포함 | 요구사항 원본은 `07` |
| API 진입점 | 요약만 포함 | 상세 요청/응답은 `04` |
| 데이터 참조 | 요약만 포함 | 테이블 기준은 `02` |
| 내부 UseCase | 요약만 포함 | 도메인 경계와 UseCase 기준은 `03` |
| 화면 연결 | 요약만 포함 | 화면별 상태는 `06` |

## 2. 기능 공통 규칙

| 항목 | 기준 |
| --- | --- |
| 외부 API | `/api/v1/**` prefix를 사용한다. |
| 내부 호출 | Java Interface로 처리하고, 다른 도메인의 Entity/Service/Repository를 직접 import하지 않는다. |
| AI 호출 | 해설·시뮬레이터 생성은 04:00 KST 배치 또는 관리자 트리거로 수행한다. 사실 기반 Q&A는 단발 요청 후 형식·정책 검증을 통과한 응답만 표시한다. |
| ANONYMOUS | 카카오 로그인 시작/콜백은 ANONYMOUS로 접근한다. |
| 로그인 필요 | 묵상 저장, 해설 본문 열람, 찬양 저장, 묵상 달력 조회는 로그인 필요다. |
| 시뮬레이터 | `READY`만 버튼 활성화, `MISSING`/`FAILED`/`DISABLED`는 비활성화한다. |
| Today QT 캐시 | 응답에는 `cache.cacheStatus`, `servedQtDate`, `staleReason`, `nextRefreshAt`을 포함한다. |
| 닉네임 나눔 | 사용자가 명시적으로 공유한 묵상만 닉네임 스냅샷과 함께 노출하고 계정 식별 정보는 응답하지 않는다. |
| 찬양 | 운영자 큐레이션과 사용자 저장만 제공하며 AI 추천, 가사·음원 저장, 직접 YouTube URL 입력은 제외한다. |
| 관리자 | `/api/v1/admin/**`는 `ADMIN` 권한이 필요하고 데이터 변경은 감사 로그 대상이다. |
| 관리자 UI 플랫폼 | 관리자 UI는 사용자 Flutter 앱과 분리된 **별도 웹 프런트엔드**다. Flutter 앱에는 관리자 메뉴를 두지 않는다(`03_아키텍처_정의서.md` v1.2 §4.9 / §13.6). |
| `@`멘션 본문 자동 삽입 | F-03/F-16 노트 본문에서 `@책 장:절` 또는 `@책 장:절-절`로 다른 성경 구절을 인용 블록으로 즉시 삽입. 트리거 기호는 `@`만 사용. 클라이언트가 입력을 파싱해 `GET /api/v1/bible/verses?bookCode=&chapter=&verseFrom=&verseTo=` 호출(`07` §6.4.1, `04` §4.2.2). |
| 닉네임 7일 변경 잠금 | 닉네임은 변경 가능하되 한 번 변경하면 7일 동안 재변경 불가. 가입 시 첫 설정과 형식 오류·중복 재설정은 잠금 대상 아님. 잠금 위반은 `409 NICKNAME_CHANGE_LOCKED`로 거절하며 응답에 `nicknameUnlockAt` 포함(`07` §F-10 닉네임 정책, `04` §4.1.5, `02` `members.nickname_last_changed_at`). |
| 성경 데이터 저장 위치 | 한글 성경 본문은 **클라이언트 로컬 SQLite + 서버 마스터 사본** (오프라인 표시 지원), 영어 성경 본문은 **서버 마스터 + 온라인 조회**. 한글은 앱 첫 실행 시 한글 번들 다운로드 API로 적재(`07` §F-01 §데이터 정책, `04` §4.2 한글 번들 다운로드, `02` `bible_books`/`bible_verses` 서버 마스터). |
| 저작권 표현 | “저작권 문제 없음”이 아니라 “저작권 리스크를 낮춘다”로 표현한다. |

## 4. 도메인별 구현 단위

| 도메인 | 기능 | 공개 UseCase 또는 API |
| --- | --- | --- |
| `domain.member` | Kakao 로그인, 토큰, 회원, 프로필, 튜토리얼 | `/oauth2/*`, `/api/v1/auth/**`, `/api/v1/me/**`, `/api/v1/members/**` |
| `domain.bible` | 성경 권/장/절 조회 | `/api/v1/bible/**` |
| `domain.qt` | QT 범위와 오늘 QT 조회 | `/api/v1/qt/**`, `/api/v1/admin/qt-passages/**` |
| `domain.study` | 승인 해설, 용어, 시뮬레이터 사용자 조회, TTS 이벤트 | `/api/v1/qt/{id}/study-content`, `/api/v1/qt/{id}/simulator-clips/**`, `/api/v1/tts/play-events` |
| `domain.note` | QT 노트, 자유 노트, 묵상 달력 | `/api/v1/notes/**`, `/api/v1/note-categories`, `/api/v1/me/meditation-calendar` |
| `domain.sharing` | 닉네임 나눔, 댓글, 좋아요 | `/api/v1/sharing-posts/**`, `/api/v1/me/sharing-posts` |
| `domain.report` | 신고 접수와 관리자 신고 처리 | `/api/v1/reports`, `/api/v1/admin/reports/**` |
| `domain.notification` | 알림과 공지 | `/api/v1/notifications/**`, `/api/v1/admin/notices/**` |
| `domain.praise` | 찬양 큐레이션과 내 찬양 목록 | `/api/v1/praise-songs/**`, `/api/v1/me/praise-songs/**`, `/api/v1/admin/praise-songs/**` |
| `domain.mission` | 마이페이지 대시보드 일부와 미션 진행 | `/api/v1/me/dashboard` 일부 |
| `domain.ai` | AI Q&A, 산출물 검증, 평가 셋, 시스템 AI 작업 | `/api/v1/ai/**`, `/api/v1/admin/ai/**`, `/api/v1/system/ai/**` |
| `domain.admin` | 관리자 대시보드와 공통 orchestration | `/api/v1/admin/dashboard` |
| `domain.audit` | 감사 로그 조회·기록 | `/api/v1/admin/audit-logs` |

모든 도메인은 단일 `qtai-server` 안에 있으며 독립 서비스로 분리하지 않는다. 관리자 API는 각 업무 도메인이 소유하고, 각 도메인의 `web/Admin{Domain}Controller.java`에서 노출된다. `domain.admin`은 대시보드와 공통 orchestration만 담당한다.

### 4.1 호출 클라이언트별 진입점

| 클라이언트 | 플랫폼 | 호출 경로 | 비고 |
| --- | --- | --- | --- |
| 사용자 앱 | Flutter (Android 우선, iOS 후순위) | `/api/v1/**` (관리자 경로 제외) + `/oauth2/**` | Today QT, 묵상, 나눔, 찬양 등 모든 사용자 기능 |
| 관리자 웹 | 별도 웹 프런트엔드 (Flutter 앱 아님) | `/api/v1/admin/**` 중심 + 일부 공통 조회 | 2026-05-19 강사님 직강 결정. SSR/SPA 채택은 v1.2 이후 결정 |
| 시스템 배치 | `SYSTEM_BATCH` 토큰 | `/api/v1/system/**` | AI 사전 생성, 검증, 평가 셋 환류 |

사용자 앱과 관리자 웹은 같은 `qtai-server`를 호출하되 호출 경로와 권한이 분리된다. 백엔드 서버를 둘 만들지 않는다.

| 기준 | 완료 조건 |
