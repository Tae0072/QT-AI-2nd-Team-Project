## 1. 목적

각 영역의 원본(SSoT)은 다음과 같다. 23번 도메인 용어사전은 이 SSoT가 정의한 내용을 새로 만들지 않고, 문서와 코드에서 같은 단어를 같은 의미로 쓰기 위한 기준만 관리한다.

| 영역 | SSoT 문서 | 책임 범위 |

## 2. 용어 사용 원칙

| 원칙 | 기준 |
| --- | --- |
| 기준 문서 우선 | 용어 해석이 충돌하면 §1에 명시한 SSoT 문서를 우선한다. 영문 코드 식별자(`enum`·테이블·컬럼·API 필드·패키지명)는 SSoT 문서의 표기를 그대로 사용한다. |
| 사용자 노출 용어 우선 | 사용자에게 보이는 용어는 쉬운 한국어를 우선한다. |
| 내부 용어와 UI 용어 분리 | 내부 검증·저장 용어와 사용자 화면 용어를 혼용하지 않는다. |
| 금지 표현 유지 | 제외된 기술이나 기능은 허용처럼 쓰지 않고, 제외·보류·금지 문맥으로만 쓴다. |
| 저작권 표현 주의 | "저작권 문제 없음"이라고 쓰지 않고, "저작권 리스크를 낮춘다"라고 쓴다. |
| 명세 없음 표기 | 02~08 SSoT에 명시되지 않은 동작·시각·정책은 임의 정의하지 않고 "확인 필요"로 표기한다. |

| 관리자 | `ADMIN` | 관리자 계정 소유 회원 | 관리자 콘솔 접근 |
| 운영 관리자 | `OPERATOR` | 운영 관리자 권한 | QT 관리, 신고 처리, 공지, 사용자 제재 |
| 검증 관리자 | `REVIEWER` | AI 산출물 검증 권한 | 산출물 목록/상세 조회, 승인, 반려, 숨김, 체크리스트 registry, 평가 케이스 관리 |
| 자체 제작자 | `CONTENT_CREATOR` | 검증용 자료와 콘텐츠 제작 메타 관리 | 검증용 주석 원문 관리 |
| 최고 관리자 | `SUPER_ADMIN` | 전체 관리자 기능 | 권한 관리 포함 |
| 시스템 배치 | `SYSTEM_BATCH` | 배치, AI 생성/검증, 내부 재처리 작업 주체 | 사용자 계정으로 처리하지 않음 |
| Access Token | Access Token | API 인증에 사용하는 JWT | 유효기간 30분 |
| Refresh Token | Refresh Token | Access Token 재발급용 토큰 | 유효기간 14일 |
| 관리자 감사 로그 | Admin Audit Log | 관리자 데이터 변경 행위 기록 | 오늘 QT, 승인 해설, AI 재생성, 찬양 큐레이션 등 필수 |

## 7. 아키텍처·API 용어

| 용어 | 표현 | 정의 | 사용 기준 |
| --- | --- | --- | --- |
| `qtai-server` | `qtai-server` | v1의 단일 Spring Boot 백엔드 애플리케이션 | Modular Monolith |
| Modular Monolith | Modular Monolith | 하나의 배포 단위 안에서 도메인을 강하게 분리하는 구조 | v1 기준 구조 |
| 외부 공개 API | External Public API | Flutter 앱, 관리자 화면, 외부 클라이언트가 호출하는 HTTP API | `/api/v1/**` prefix |
| 내부 도메인 인터페이스 | Internal Domain Interface | `qtai-server` 내부 도메인 간 호출용 Java Interface | HTTP 경로가 아님 |
| UseCase Interface | UseCase Interface | 도메인 외부에 공개하는 Java Interface | 다른 도메인은 `api/`의 UseCase/DTO만 호출 |
| DTO | Data Transfer Object | 도메인 간 또는 API 응답에 사용하는 전달 객체 | Entity 직접 노출 금지 |
| `domain.member` | Member/Auth | Kakao 로그인, 토큰, 회원, 프로필, 튜토리얼 담당 도메인 | `POST /api/v1/auth/kakao`, `/api/v1/me/**` |
| `domain.bible` | Bible | 성경 권/장/절 조회 담당 도메인 | `/api/v1/bible/**` |
| `domain.qt` | QT | QT 범위와 오늘 QT 조회 담당 도메인 | `/api/v1/qt/**` |
| `domain.study` | Study Content | 승인 해설, 용어, 시뮬레이터 사용자 조회, TTS 이벤트 담당 도메인 | 미승인 산출물 노출 금지 |
| `domain.note` | Note | QT 노트, 자유 노트, 묵상 달력 담당 도메인 | 자동 저장 금지 |
| `domain.sharing` | Sharing | 닉네임 나눔, 댓글, 좋아요 담당 도메인 | 공유글 스냅샷 유지 |
| `domain.report` | Report | 신고 접수와 관리자 신고 처리 담당 도메인 | 대상 도메인 Repository 직접 변경 금지 |
| `domain.notification` | Notification | 알림과 공지 담당 도메인 | 중복 알림 방지 |
| `domain.praise` | Praise | 찬양 큐레이션과 내 찬양 목록 담당 도메인 | 가사·음원 저장 금지 |
| `domain.ai` | AI | AI Q&A, 산출물 검증, 평가 셋, 시스템 AI 작업 담당 도메인 | 다중 턴 챗봇/SSE 금지 |
| `domain.admin` | Admin | 관리자 대시보드와 공통 orchestration 담당 도메인 | 전체 관리자 API 단독 소유 금지 |
| `domain.audit` | Audit | 감사 로그 조회·기록 담당 도메인 | 수정/삭제 API 금지 |

## 8. 이벤트·운영 용어

| 용어 | 표현 | 정의 | 사용 기준 |
| --- | --- | --- | --- |
| 인프로세스 이벤트 | In-process Event | 같은 애플리케이션 내부에서 발행·구독하는 이벤트 | v1 이벤트 방식 |
| `ApplicationEventPublisher` | Spring `ApplicationEventPublisher` | v1에서 사용하는 Spring 이벤트 발행 도구 | Kafka 대체가 아니라 v1용 내부 이벤트 |
| 이벤트 핸들러 | Event Handler | 이벤트를 받아 후속 처리를 수행하는 구성요소 | 실패 로그 기록 필요 |
| `eventId` | `eventId` | 이벤트 식별자 | 실패 로그와 재처리 기준 |
| 핸들러 실패 로그 | Handler Failure Log | 이벤트 핸들러 실패 시 남기는 로그 | 재처리 근거 |
| 재처리 가능 | Reprocessable | DB 상태와 로그를 기준으로 관리자 또는 배치가 다시 처리 가능한 상태 | "유실률 0%" 보장 표현 금지 |
| 오늘 QT 캐시 | Today QT Cache | 오늘 QT 응답 성능을 위한 캐시 | 04:00 KST 사용자 노출/cache refresh 기준으로 교체 |
| 배치 | Batch Job | 정해진 시각 또는 관리자 트리거로 실행되는 백엔드 작업 | 사용자 요청 경로가 아님 |
| 관리자 트리거 | Admin Trigger | 관리자가 특정 본문의 재생성을 요청하는 운영 기능 | 실제 실행은 `SYSTEM_BATCH` 작업으로 기록 |
