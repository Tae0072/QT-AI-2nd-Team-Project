## 1. 문서 목적과 경계

이 문서는 구현 저장소에서 첫 PR을 만들기 전에 반영해야 할 기준을 작업 가능한 체크리스트로 정리한다.

요구사항 원본은 `07_요구사항_정의서.md`가 책임지고, 아키텍처 상세는 `03_아키텍처_정의서.md`, API 계약은 `04_API_명세서.md`, 협업 규칙은 `09_Git_규칙.md`, 품질 게이트는 `18_코드_품질_게이트.md`, 용어 기준은 `23_도메인_용어사전.md`가 책임진다.

이 문서 저장소에는 실제 Spring Boot/Flutter 골격, OpenAPI 파일, DB/데이터 디렉터리, infra 디렉터리, CI workflow 파일을 두지 않는다. 아래 항목은 별도 구현 GitHub에서 만들 산출물의 기준이다.

| 구분 | 이 문서에서 관리 | 다른 문서에서 관리 |
| --- | --- | --- |
| 구현 저장소 초기 세팅 항목 | 포함 | 세부 설계 원칙은 `03` |
| PR 템플릿 반영 항목 | 포함 | 원문 템플릿은 `14` |
| CI/Guard 반영 순서 | 포함 | 상세 게이트 정의는 `18` |
| 패키지·API·데이터 기준 요약 | 포함 | 상세 기준은 `03`, `04`, `07`, `23` |
| 기능 요구사항 본문 | 포함하지 않음 | `07` |

문서 간 충돌이 발생하면 기능 포함 여부와 사용자 노출 기준은 `07_요구사항_정의서.md` v3.5를 우선한다.

| 체크 | 기준 |
| --- | --- |
| 단일 백엔드 | `qtai-server` 하나만 둔다. `auth-service`, `bible-service`, `note-service` 같은 독립 서비스는 만들지 않는다. |
| 도메인 경계 | 다른 도메인의 Entity, Service, Repository 직접 import를 금지한다. |
| 도메인 내부 구조 | 각 도메인은 `api/`, `internal/`, `web/`을 기본으로 둔다. 다른 도메인 의존이 필요한 경우에만 `client/{domain}/` 아래 테스트 대역 또는 어댑터를 둔다. |
| 공개 범위 | 다른 도메인이 import 가능한 타입은 `api/` 하위 UseCase/DTO로 제한한다. |
| 노트/찬양 | `domain.note`, `domain.praise` 기준으로 두고, 공유는 `domain.sharing`이 스냅샷을 소유한다. |
| `admin` | v1은 단일 `qtai-server` 내부 지원 모듈 · v2 MSA에서 코어(mypage/platform) 서비스에 귀속(2026-06-08) |
| 관리자 UI 플랫폼 | 관리자 UI는 사용자 Flutter 앱과 분리된 **별도 웹 프런트엔드**(`admin-web/`)에 둔다. Flutter 앱에 관리자 화면(AD-01~AD-08)을 포함하지 않는다. SSR/SPA 채택은 v1.2 이후 결정(`03_아키텍처_정의서.md` v1.3 §4.9 / §13.6). |
| 관리자 API 노출 위치 | 관리자 API는 각 업무 도메인의 `web/Admin{Domain}Controller.java`에 둔다. `/api/v1/admin/**` 전체를 `domain.admin`이 단독 소유하지 않는다. |
| `audit` | 횡단 관심사로 두며, 다른 도메인이 `audit.api.WriteAuditLogUseCase`를 직접 호출할 수 있다. |
| 외부 API | Flutter/Admin이 호출하는 HTTP API는 `/api/v1/**`로 통일한다. |
| 내부 호출 | 내부 도메인 호출은 Java Interface로 표현하고 OpenAPI에 노출하지 않는다. |
| 주요 UseCase | `LoginUseCase`, `GetMemberUseCase`, `GetBibleVerseUseCase`, `GetQtUseCase`, `CreateNoteUseCase`, `CreateShareUseCase`, `SendNotificationUseCase`, `GenerateAiResponseUseCase`, `WriteAuditLogUseCase`처럼 기능 단위 Interface 기준을 따른다. |
| 작업 기록 | 각 개발자는 `workspaces/{담당자}/workflows`에 작업 전 계획, `workspaces/{담당자}/reports`에 작업 후 리포트를 남긴다. |

## 4. PR 템플릿 반영 체크리스트

`09_Git_규칙.md`의 PR 템플릿을 구현 저장소 `.github/pull_request_template.md`에 반영한다.

| 체크 | 필수 내용 |
| --- | --- |
| 관련 요구사항 | 기능 PR은 `07`의 F-ID 또는 관련 문서 링크를 포함한다. |
| 변경 유형 | `feat`, `fix`, `refactor`, `test`, `docs`, `chore` 중 선택한다. |
| 도메인 경계 | 직접 import 금지, DTO/Interface 통신 여부를 확인한다. |
| 금지 기술 | RAG, ChromaDB, vector DB, Elasticsearch v1 도입이 없음을 확인한다. (Kafka·Kubernetes·Helm은 v2 MSA 분리 트랙 허용 — 2026-06-08) |
| 세션형 AI | 자유 챗봇, 다중 턴 대화, SSE, `/ai/sessions/**`가 없고 F-15 Q&A는 단발·검증 흐름인지 확인한다. |
| Today QT | 본문, 해설 진입점, 묵상 진입점, 시뮬레이터 상태값, `cache.cacheStatus`를 반환하는지 확인한다. |
| 인증 | `/api/v1/auth/**`, `/api/v1/me` 외 교회 인증 경로가 없는지 확인한다. |
| 닉네임 나눔 | 닉네임 스냅샷 노출, 계정 식별 정보 비노출, 명시적 공유, 신고 관리자 전달을 확인한다. |
| 찬양 | AI 추천, 가사·음원 저장, 직접 YouTube URL 입력 API가 없음을 확인한다. |
| 성경 데이터 | repo URL, license, translationName, attribution, redistributionAllowed를 확인한다. |
| 권한 | 관리자 API는 기능별 관리자 권한, 배치/AI 내부 작업은 `SYSTEM_BATCH` 기준을 확인한다. |
| 관리자 UI 분리 | Flutter 앱(`flutter-app/`) PR에 관리자 화면이 포함되지 않았는지 확인한다. 관리자 UI 변경은 `admin-web/`(또는 합의된 경로) PR에서 처리한다. |
| 테스트 | 단위 테스트, 통합 테스트, 미작성 사유를 명시한다. |

### 4.1 개발자별 작업 기록 폴더 체크리스트

## 5. CI/Guard 반영 체크리스트

`18_코드_품질_게이트.md`의 GitHub Actions 예시는 구현 저장소 구조에 맞게 경로만 조정해서 반영한다.

| 게이트 | 구현 저장소 반영 기준 | 실패 시 |
| --- | --- | --- |
| Spectral lint | `apis/*/openapi.yaml` 검사 | 머지 차단 |
| Backend build/test | Java 21, Spring Boot 3.3 기준 `./gradlew -p qtai-server build` | 머지 차단 |
| Coverage | Jacoco 70% 이상 | 머지 차단 |
| Modulith/ArchUnit | 도메인 경계와 금지 import 검사 | 머지 차단 |
| Requirements Guard | 금지 기술, 금지 API, 금지 데이터 검사 | 머지 차단 |
| Document/Terminology Guard | 문서·용어 금지 표현 검사 | 머지 차단 또는 문맥 확인 |
| Flutter analyze/test | `flutter-app` 존재 시 실행 | 머지 차단 |
| Secret scan | gitleaks | 머지 차단 + Lead 알림 |

### 5.1 Requirements Guard 필수 검사

| 검사 대상 | 금지 또는 차단 기준 |
| --- | --- |
| 세션형 AI API | `/ai/sessions/**`, SSE, 다중 턴 챗봇. 해설·시뮬레이터 사용자 요청 경로 생성 호출 |
| 검색·임베딩 | RAG, ChromaDB, vector DB, Elasticsearch |
| 메시징·인프라 | Kafka, Kubernetes, Helm: v1 단일배포 금지 · v2 MSA 허용(2026-06-08) |
| 성경 데이터 | 개역개정, ESV, NIV |
| 교회 인증 | 회원가입 화면 버튼, API, `churchAuth`, `church_cert` |
| 찬양 | AI 찬양 추천, 가사·음원 저장, 사용자 직접 YouTube URL 입력 |
| 닉네임 나눔 | `sharing-posts` 응답의 userId, email, profile 등 계정 식별 정보 |
| Today QT 캐시 | `cacheStatus` 누락, 00:00~04:00 구간 새 캐시 강제 조회 |

Requirements Guard는 구현 코드, API 산출물, DB seed, CI workflow를 대상으로 한다. `.github/pull_request_template.md`는 금지 항목을 체크리스트로 설명하는 문서성 파일이므로 금지 패턴 스캔 대상에서 제외하고, `.github/workflows`만 검사한다.

### 5.2 Document/Terminology Guard 필수 검사

| 피해야 할 표현 | 권장 표현 |
| --- | --- |
| 저작권 문제 없음 | 저작권 리스크를 낮춘다 |
| 이벤트 유실률 0% 보장 | 핸들러 실패 로그 + 재처리 가능 |
| 내부 API 경로 | 내부 Java Interface |
| AI 찬양 추천 | 운영자 사전 큐레이션 |
| 오늘 QT 100% = 모든 클립 있음 | 시뮬레이터 상태값 100% 반환 |
| 캐시 실패 = 장애 | `STALE_FALLBACK` 또는 `EMPTY` 상태로 명시 |

## 6. 백엔드 첫 PR 체크리스트

첫 백엔드 PR은 기능을 크게 만들기보다 패키지 경계와 테스트 골격을 먼저 만든다.

| 순서 | 작업 | 완료 기준 |
| --- | --- | --- |
| 1 | `qtai-server` Gradle/Spring Boot 골격 생성 | Java 21, Spring Boot 3.3, 테스트 실행 가능 |
| 2 | 도메인 패키지 기준 반영 | `domain.member`, `domain.bible`, `domain.qt`, `domain.study`, `domain.note`, `domain.sharing`, `domain.report`, `domain.notification`, `domain.praise`, `domain.mission`, `domain.ai`, `domain.admin`, `domain.audit` 기준 반영 |

## 7. API 첫 PR 체크리스트

API 산출물은 외부 공개 HTTP API만 OpenAPI로 관리한다.

| 체크 | 기준 |
| --- | --- |
| Prefix | 모든 외부 공개 API는 `/api/v1`로 시작한다. |
| 인증 | `POST /api/v1/auth/kakao`, `POST /api/v1/auth/token/refresh`, `POST /api/v1/auth/logout`, `GET /api/v1/me` 기준을 반영한다. |
| ANONYMOUS | 로그인 시작과 콜백은 `ANONYMOUS` 기준이다. |
| 로그인 필요 | 묵상 저장, 해설 본문 열람, 찬양 저장, 묵상 달력 조회는 `USER`이다. |
| 관리자 | `/api/v1/admin/**`은 `ADMIN`이다. |
| Today QT | 해설 본문을 직접 싣지 않고 해설 진입점, 묵상 진입점, `cache.cacheStatus`를 제공한다. |
| Simulator | 상태는 `READY`, `MISSING`, `FAILED`, `DISABLED` 중 하나다. |
| 닉네임 나눔 | `/api/v1/notes/{noteId}/share`, `/api/v1/sharing-posts/**`, `/api/v1/me/sharing-posts` 계약을 반영하고 계정 식별 정보는 제외한다. |
| 찬양 | `/api/v1/praise-songs`, `/api/v1/me/praise-songs/**`, `/api/v1/admin/praise-songs/**`만 반영하고 직접 URL 입력 API는 만들지 않는다. |
| 관리자 상세 | dashboard, qt-passages, ai, reports, praise-songs, audit-logs 범위를 반영한다. |
| 내부 Interface | `CreateNoteUseCase`, `CreateShareUseCase`, `ListPraiseUseCase`, `WriteAuditLogUseCase` 같은 내부 Java Interface는 OpenAPI에 포함하지 않는다. |
