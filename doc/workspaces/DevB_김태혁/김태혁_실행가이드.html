김태혁 — 시뮬레이터 상태 모델, 클립 조회, 실패·누락 처리 (study 도메인)
1 Today QT 100%는 시뮬레이터 상태값 정상 응답을 뜻하며 모든 클립 필수 생성이 아니다.
2 READY만 버튼 활성화, MISSING/FAILED/DISABLED는 비활성화한다.
3 본문 범위별 클립은 재사용하고 사용자 요청마다 재생성하지 않는다.
4 study 도메인은 domain.bible/domain.ai 내부 클래스를 직접 import하지 않고 Interface/DTO로만 연동한다.
5 시뮬레이터 API는 GET /api/v1/qt/{qtPassageId}/simulator로 단순화한다. 클라이언트는 clipId를 직접 지정하지 않고 서버가 APPROVED 상태 clip을 자동 선택한다.
본인 우선 액션 4가지
1 상태 enum과 UI 버튼 매핑 기준을 먼저 고정
2 Today QT 응답에 simulatorStatus가 항상 포함되는지 확인
3 실패 로그와 재처리 가능 기준을 report에 남김
4 READY와 비활성 상태를 모두 시연 샘플로 준비
