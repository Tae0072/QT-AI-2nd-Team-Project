## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| QT 본문 시각 | 성서 유니온 공개 00:00 KST, 우리 시스템 수집 배치 04:00 KST |
| 성경 데이터 | Repo URL, 라이선스, 번역본명, 출처 표기 방식, 재배포 가능 여부가 확인된 GitHub 공개 JSON만 사용 |
| 금지 번역본 | 개역개정, ESV, NIV |
| 저장 범위 | 성서 유니온에서는 본문 텍스트가 아니라 본문 범위 정보만 수집 |
| Today QT | 본문·해설 진입점·묵상 진입점·시뮬레이터 상태값을 포함해 응답 |
| 도메인 경계 | bible 도메인은 domain.ai/domain.study 내부 클래스를 직접 import하지 않고 DTO 또는 Interface로만 통신 |

## 5. 브랜치·PR·워크플로우 기록 규칙

| 항목 | 이지윤 기준 |
| --- | --- |
| 브랜치 생성 | `dev` 최신화 후 `feature/bible-*`, `bugfix/bible-*`, `test/bible-*` 형식으로 생성 |
| PR 대상 | 항상 `dev` |
| 자동 PR | 브랜치 push 후 GitHub PR을 생성하면 CI와 자동 리뷰가 실행된다. 자동 머지는 하지 않는다. |
| 작업 폴더 | 별도 구현 GitHub에서 `workspaces/이지윤/workflows`, `workspaces/이지윤/reports`를 사용 |
| 기록 순서 | 작업 전 workflow 작성 → 작업·검증 → report 작성 → PR 생성 |
| 기준 문서 | 상세 명령과 템플릿은 `개발자별_일정표/00_공통_브랜치_PR_워크플로우_규칙.md`를 따른다. |
