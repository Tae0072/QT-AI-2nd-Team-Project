## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| QT 본문 시각 | 성서 유니온 공개 00:00 KST, 우리 시스템 수집 배치 04:00 KST |
| 성경 데이터 | Repo URL, 라이선스, 번역본명, 출처 표기 방식, 재배포 가능 여부가 확인된 GitHub 공개 JSON만 사용 |
| 금지 번역본 | 개역개정, ESV, NIV |
| 저장 범위 | 성서 유니온에서는 본문 텍스트가 아니라 본문 범위 정보만 수집 |
| Today QT | 본문·해설 진입점·묵상 진입점·시뮬레이터 상태값을 포함해 응답 |
| 도메인 패키지 | `domain.bible`은 `api/internal/web` 3분할(다른 도메인 호출이 없어 `client/` 없음). `internal/`은 외부 접근 절대 금지(03 v1.2 §3.1). |
| 도메인 경계 | bible 도메인은 domain.ai/domain.study 내부 클래스를 직접 import하지 않고 `api/UseCase`로만 통신 |
| 한글·영어 저장 위치 | 한글 본문은 **클라이언트 로컬 SQLite + 서버 마스터 사본**, 영어 본문은 **서버 마스터 + 온라인 조회만**. 한글 번들은 `GET /api/v1/bible/bundle?language=ko`로 첫 적재(07 v3.4 §F-01, 04 v1.6 §4.2.2.1, 02 v2.2). |
| 범위 조회 | `GET /api/v1/bible/verses?bookCode=&chapter=&verseFrom=&verseTo=` 범위 조회 지원. `@`멘션 본문 자동 삽입 흐름(F-03/F-16)에서 사용. 단일 인용 범위 상한은 후속 정책(04 v1.4 §4.2.2). |
| 한글 번들 버전 관리 | 번역본 갱신 시 `bundleVersion`을 올리고, 클라이언트는 `version` 파라미터로 `304 Not Modified` 또는 재다운로드 판단(04 v1.6 §4.2.2.1). |

## 6. PR 전 체크

| 체크 | 기준 |
| --- | --- |
| 출처 | 성경 JSON 출처 표가 비어 있지 않음 |
| 금지 데이터 | 개역개정, ESV, NIV 데이터나 문구가 들어가지 않음 |
| 본문 범위 | 하루 1개 본문 범위가 다중 장/다중 절을 표현할 수 있음 |
| Today QT | 00:00/04:00 기준과 캐시 정책을 깨지 않음 |
| 경계 | domain.ai/domain.study 클래스 직접 import 없음. `domain.bible.internal/`은 외부 접근 차단 |
| 범위 조회 | `verseFrom`/`verseTo` 파라미터가 정상 동작 (단일 절·범위 모두 지원) |
| 한글 번들 다운로드 | `GET /api/v1/bible/bundle?language=ko` 응답에 `bundleVersion` 포함, `version` 일치 시 `304 Not Modified` |
| 영어 번들 | v1에서 영어 번들 다운로드 API는 제공하지 않음 (영어는 온라인 조회만) |
