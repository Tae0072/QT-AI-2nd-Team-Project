# 스터디노트 — qt·study 도메인 이전, 쉽게 풀어보기 (2026-06-10)

> 입문자용 정리. "오늘 무슨 일을 왜 했는지"를 쉬운 말로 적었다.

## 0. 한 줄 요약

큰 서버(모놀리식) 안에 있던 **"오늘의 QT"(qt)** 와 **"해설·시뮬레이터"(study)** 기능을,
새로 만드는 작은 서버 모듈(`service-bible`)로 **옮겨 심었다.** 원본은 그대로 두고 복사해서
옮기는 방식이라 안전하다.

## 1. 핵심 개념

### Strangler(스트랭글러) 방식
큰 건물을 한 번에 부수지 않고, 옆에 새 건물을 지어 방을 하나씩 옮기는 방식. 모놀리식을 그대로
둔 채 도메인을 하나씩 새 모듈로 옮긴다. 옮긴 게 잘 돌면 나중에 원본을 지운다.

### 도메인 경계: api / internal / web / client
한 도메인(예: qt)은 4개의 방으로 나뉜다.
- **api**: 다른 도메인에 "이건 써도 돼"라고 공개하는 계약(인터페이스 + DTO).
- **internal**: 진짜 속살(Entity, Service, Repository). **다른 도메인이 절대 직접 못 본다.**
- **web**: 바깥(앱)에서 들어오는 HTTP 입구(Controller).
- **client**: 다른 도메인이나 외부 시스템을 부를 때 쓰는 어댑터.

규칙: 도메인끼리 부를 때는 반드시 상대의 **api**만 본다. 상대의 **internal**을 직접 import하면
경계가 무너진다. 이걸 자동으로 막는 게 아래 ArchUnit 테스트.

### Mock(임시 가짜 구현)
qt는 "노트(note)" 기능을 살짝 쓴다("오늘 쓴 묵상노트 있어?"). 그런데 note는 **다른 서버**라
지금은 진짜로 못 부른다. 그래서 "항상 없다고 답하는 가짜"(`GetNoteUseCaseMock`)를 끼워 둔다.
나중에 진짜 연결이 생기면 이 가짜 파일을 지운다. (CLAUDE.md §4 규칙)

### 표준 페이징 envelope (`PageResponse`)
목록을 페이지로 나눠 줄 때, 스프링 기본 `Page`를 그대로 내보내면 내부 구조가 그대로 노출돼
스프링 버전이 바뀌면 앱이 깨질 수 있다. 그래서 **필요한 정보만 담은 우리 봉투**(content, page,
size, totalElements, totalPages, first, last)로 바꿔 내보낸다.

### denyAll 회귀 방지
`/api/v1/admin/**`(관리자 경로)는 이 콘텐츠 서버에서 **무조건 막는다(denyAll)**. 관리자 기능은
별도 admin-server가 담당하기 때문. "관리자 토큰만 있으면 열리는" 실수를 막기 위해, 관리자
권한이 있어도 403으로 막히는지 테스트로 못 박았다.

### ArchUnit(아키텍처 테스트)
"코드가 규칙을 지키는지"를 코드로 검사하는 도구. 여기선 "어떤 도메인도 다른 도메인의 internal을
import하지 않는다"를 자동 검사한다. 누가 실수로 경계를 넘으면 빌드가 빨갛게 실패한다.

## 2. 오늘 부딪힌 함정 2가지 (기록해 둘 것)

1. **마운트가 파일을 잘라서 보여줬다.** 작업용 리눅스 환경(샌드박스)이 일부 원본 파일을 중간에서
   잘린 채로 보여줬다. 그걸 복사하니 컴파일이 "파일이 갑자기 끝났다"며 실패. → **파일 복사는
   실제 컴퓨터(호스트)에서 직접** 해야 안전(robocopy). 빌드를 돌리면 잘린 파일은 컴파일 에러로
   바로 들통나니, "빌드 통과 = 파일 멀쩡"이라고 믿어도 된다.
2. **IDE가 빌드 폴더를 잠갔다.** 에디터가 `build` 폴더를 붙들고 있어 테스트가 폴더 삭제에서
   실패. → 잠긴 `test-results` 폴더만 지우고 다시 빌드.

## 3. 왜 이렇게까지 테스트를 많이?

PR #1 때 자동 리뷰어가 "다음 PR에서는 이런 것도 챙겨라"라고 숙제를 남겼다(통합테스트, 인증
헬퍼, 표준 페이징, 경계 테스트). 그 숙제를 이번 PR에서 같이 끝내야 리뷰어가 한 번에
**APPROVE(승인)** 하고 자동 머지된다. 그래서 첫 제출부터 높은 품질로 맞췄다.

빌드 통과, 테스트 32개 전부 초록불. qt·study가 새 모듈에서 정상 동작하고, 경계 규칙도 지켜진다.
