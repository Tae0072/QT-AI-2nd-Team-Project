# 스터디 노트 — "모놀리식 복사" 어드민 서버 (입문자용)

> 2026-06-10, Day3 admin-server. 사용자 서비스는 잘게 쪼갰는데, 관리자는 왜 통째로 복사했나.

## 1. 왜 admin은 MSA로 안 쪼개고 "모놀리식 복사"인가
관리자 기능은 ① 트래픽이 거의 없고 ② 보통 1명이 쓰고 ③ 여러 도메인을 가로질러 CRUD(성경 오타수정·신고관리·AI 검수)한다. 이런 걸 마이크로서비스로 쪼개면 서비스 간 호출만 늘어 손해다. 그래서 **모놀리식 원본을 통째 복사**해서 admin 전용 서버로 두고, **같은 DB**에 붙여 내부 서비스를 바로 호출한다. "MSA의 목적은 장애 격리지, 무조건 쪼개는 게 아니다"의 실천.

## 2. "통째 복사 후 컨트롤러만 정리"의 의미
강사님 방식 = 모놀리식 `src/main`을 그대로 복사 → **HTTP 컨트롤러 중 `/api/v1/admin/**`만 남기고 나머지(사용자용·system) 삭제**. 폴더·서비스·엔티티·로직은 그대로 둔다.
- 왜 컨트롤러만? 컨트롤러는 "외부에 노출되는 입구"다. 입구만 admin용으로 막으면, 내부 서비스(리포트 생성 등)는 admin 컨트롤러가 그대로 호출할 수 있다.
- 서비스/리포지토리를 안 지우는 이유: admin 기능이 그것들을 쓰고, 지우면 컴파일이 줄줄이 깨진다. "입구만 정리, 내부는 보존."

## 3. 멀티모듈에서 "같은 패키지"가 충돌 안 하는 이유
admin-server는 자체 `com.qtai.common`을 갖고, lib-common에도 `com.qtai.common`이 있다. 같은 패키지명인데 왜 안 부딪힐까? **Gradle 멀티모듈에서 각 모듈은 독립된 컴파일 단위(별 jar)**라, admin-server가 lib-common에 의존하지 않으면 두 common은 서로 다른 모듈에 격리돼 클래스패스에서 만나지 않는다. 그래서 admin-server는 lib-common 비의존 standalone으로 두었다.

## 4. 복사한 모놀리식을 H2에서 부팅시키기 — 테스트 하니스 차용
모놀리식은 운영 설정(Flyway MySQL 마이그레이션 ON, 실제 DB)이라 그대로 H2 테스트에 올리면 깨진다. 모놀리식이 **이미 갖고 있는 테스트 하니스**를 그대로 가져오면 된다:
- `application-test.yml`: H2(MySQL 모드) + `ddl-auto: create-drop`(엔티티로 스키마 생성) + `flyway.enabled: false`.
- `JwtTestKeysContextCustomizerFactory` + `META-INF/spring.factories`: 평문 키를 저장소에 안 두고 **테스트 부팅 때 RSA 키를 런타임 생성**해 주입.
포인트: **테스트 본체(수백 개)는 복사하지 않는다** — 삭제한 컨트롤러의 테스트가 깨지기 때문. 부팅·노출면·보안 최소 테스트만 새로 쓴다.

## 5. "지운 게 지워졌다"를 테스트로 못박기
컨트롤러 노출면 테스트: 클래스패스를 스캔해 `@RestController`를 모두 찾고, **매핑 경로가 전부 `/api/v1/admin`으로 시작하는지** 검사한다. 누군가 실수로 사용자/system 컨트롤러를 되살리면 빌드가 깨진다. "있어야 할 것"뿐 아니라 **"없어야 할 것이 없음"도 테스트**한다.

## 6. 거버넌스: 가드를 풀려면 SSoT부터
로컬 k8s 배포를 위해 CI 가드(k8s 금지)를 풀어야 했는데, 리뷰가 막았다 — **CLAUDE.md(SSoT)가 여전히 "K8s 금지"인데 CI만 먼저 풀 수 없다**(우선순위 07→03→...→CI). 배경음악 예외 때처럼 ① SSoT(CLAUDE.md §8)에 예외 명시 ② Lead 결정 문서 추가 ③ PR 본문에 근거 링크 — 이 순서를 지켜야 가드 완화가 통과된다. "규칙을 바꾸려면 규칙의 원본부터 바꾼다."
