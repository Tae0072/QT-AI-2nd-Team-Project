# 2026-06-09 · admin-web 관리자 권한 조회 오류 처리 스터디 노트

## 왜 오류를 나눠야 하나

관리자 웹은 로그인 후 `GET /api/v1/admin/me`로 세부 관리자 권한을 확인한다. 이 요청 실패를 모두 로그아웃으로 처리하면, 백엔드 재기동이나 일시적 네트워크 오류만으로 사용자가 토큰을 잃는다.

## 분기 기준

- `401`: 토큰 만료 또는 인증 실패. 저장 토큰 삭제 후 로그인 화면으로 보낸다.
- `403`: ADMIN 또는 admin_users 세부 권한 부족. 저장 토큰 삭제 후 로그인 화면으로 보낸다.
- 그 외 네트워크/5xx/timeout: 토큰은 유지하고 권한 확인 실패 화면에서 재시도를 제공한다.

## 보안 관점

프론트는 UX 방어선이다. 실제 권한 검증은 백엔드가 계속 수행한다. 네트워크 오류에서 토큰을 유지하더라도 `/api/v1/admin/**` 호출은 백엔드 인증/인가를 반드시 통과해야 한다.
