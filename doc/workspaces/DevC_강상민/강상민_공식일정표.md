## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| AI 호출 | 해설·시뮬레이터는 04:00 KST 배치 또는 관리자 트리거로만 생성한다. F-15 사실 기반 Q&A는 단발·검증 흐름으로만 실행한다. |
| 세션형 AI | AI 자유 챗봇, 다중 턴 대화, SSE, `/ai/sessions/**` 사용자 경로는 만들지 않는다. |
| 해설 구조 | `commentary_materials` 기반으로 AI가 재해석한 산출물을 검증 후 `verse_explanations`로 노출한다. |
| 검증 참조 원문 | 최신 주석 검증 참조 원문은 검증용 비공개 데이터이며 사용자에게 노출하지 않는다. |
| 검증 | 편집자 에이전트가 사실 정합성과 말투를 검증하고 통과한 결과만 `verse_explanations`에 승인 상태로 연결한다. |
| 실패 처리 | 실행 실패는 `ai_generation_jobs`, `ai_generated_assets`, `ai_validation_logs`에 상태와 사유를 기록하고 재처리 가능하게 둔다. |

## 5. 브랜치·PR·워크플로우 기록 규칙

| 항목 | 강상민 기준 |
| --- | --- |
| 브랜치 생성 | `dev` 최신화 후 `feature/ai-*`, `bugfix/ai-*`, `test/ai-*` 형식으로 생성 |
| PR 대상 | 항상 `dev` |
| 자동 PR | 브랜치 push 후 GitHub PR을 생성하면 CI와 자동 리뷰가 실행된다. 자동 머지는 하지 않는다. |
| 작업 폴더 | 별도 구현 GitHub에서 `workspaces/강상민/workflows`, `workspaces/강상민/reports`를 사용 |
| 기록 순서 | 작업 전 workflow 작성 → 작업·검증 → report 작성 → PR 생성 |
| 기준 문서 | 상세 명령과 템플릿은 `개발자별_일정표/00_공통_브랜치_PR_워크플로우_규칙.md`를 따른다. |

## 6. PR 전 체크

| 체크 | 기준 |
| --- | --- |
| 해설·시뮬레이터 경로 | 사용자 요청에서 즉시 생성용 LLM 호출 없음 |
| SSE | SSE 스트리밍 코드나 `/ai/sessions/**` 사용자 API 없음 |
