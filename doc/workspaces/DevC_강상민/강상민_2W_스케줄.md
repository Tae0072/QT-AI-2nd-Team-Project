다음 작업:

OpenAPI 정합화, 관리자 AI 산출물 조회 API, 체크리스트 버전 관리 API는 완료했다. 다음 작업은 확정된 우선순위에 따라 `validation_reference_jobs` 생성/조회/만료 API부터 진행한다.
Lead 확인이 필요한 세부 항목과 확정 완료 항목은 `doc/workspaces/DevC_강상민/reports/checkList.md`에 모아 추적한다.

구현 PR 후보:

1. `validation_reference_jobs` 생성/조회/만료 API를 별도 PR에서 구현한다. 검증 참조 원문 전체를 노출하지 않고 참조 작업 메타데이터와 상태만 관리한다.
2. 로그인/인증 담당 의존 작업: `service_accounts` 기반 시스템 토큰 검증 필터와 `/api/v1/system/**` 전역 보안 설정 구현.
   DevC는 후속 PR merge 전까지 `/api/v1/system/ai/**` 컨트롤러에서 `SYSTEM_BATCH`/`ROLE_SYSTEM_BATCH` authority를 확인하는 최소 방어선을 유지한다.

확정/정리 요약:

- 관리자 AI 산출물 조회는 `OPERATOR`를 차단하고 `ADMIN + REVIEWER/SUPER_ADMIN`만 허용한다.
- `ai_validation_checklist_versions` 관리 API는 `validation_reference_jobs` API보다 먼저 구현 완료했다.
- `inputHash` 컬럼/unique key 확장은 W2/MVP 범위에서 제외한다. 중복 방지는 현재 구현된 `QUEUED`/`RUNNING` 진행 중 job 차단 기준을 유지한다.
- `SUMMARY`/`GLOSSARY`는 독립 generation job type으로 열지 않고 `AiGeneratedAssetType`으로만 유지한다.

계속 확인 필요:

- 저장소 표준 `.spectral.yaml` ruleset 위치를 확정하거나 ruleset 파일을 추가한다.

## 같은 sprint 후속 처리 보장

후속 PR이 merge되기 전까지 `/api/v1/system/ai/**` API는 `SYSTEM_BATCH`/`ROLE_SYSTEM_BATCH` authority를 컨트롤러에서 확인하는 최소 방어선으로 동작합니다. OpenAPI 계약은 현재 구현 기준으로 반영됐으며, service account 기반 전역 보안 게이트는 공통 보안 후속 작업으로 분리합니다.
