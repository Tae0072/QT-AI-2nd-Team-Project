# Report - 2026-06-12 AI 산출물 재생성 진행 중 표시

## 요약

숨김/반려 산출물에서 재생성 요청이 이미 들어간 뒤 같은 상세에서 다시 `재생성`을 누르면, 서버가 중복 active job 때문에 거절하면서 사용자가 “재생성이 실패했다”고 오해할 수 있는 문제를 수정했다.

재생성 요청이 성공하거나, 이미 진행 중인 job 때문에 상태 전이 오류가 반환되면 같은 상세 Drawer에서 `재생성` 버튼 대신 `재생성 작업 진행 중` 표시를 보여주도록 변경했다.

## 확인한 근거

- 서버 `AiService`는 같은 대상의 `QUEUED/RUNNING` 생성 작업이 있으면 재생성 요청을 거절한다.
- 상세 API 응답에는 현재 대상의 active regeneration job 요약 필드가 없다.
- 브라우저에서 반려 산출물 `#31102`에 재생성 요청을 다시 보내면 공통 메시지 `상태 전이를 수행할 수 없습니다.`가 표시되고 버튼은 그대로 남아 있었다.

## 변경 내용

- `admin-web/src/pages/AiAssetsPage.tsx`
  - `RegenerationJobNotice` 타입을 추가했다.
  - `regenerationJobsByAssetId` 상태를 추가했다.
  - 재생성 성공 시 `generationJobId/status`를 해당 산출물 ID에 저장한다.
  - 재생성 요청 중 상태 전이 오류가 반환되면 해당 산출물을 `QUEUED/RUNNING` 진행 중 상태로 표시한다.
  - 진행 중으로 표시된 산출물 상세에서는 `재생성` 버튼 대신 `재생성 작업 진행 중` Tag를 보여준다.

## 유지한 내용

- 서버 API, DTO, 중복 job 정책은 변경하지 않았다.
- 재생성 요청 payload와 모달 입력 방식은 유지했다.
- 승인/검증중 산출물에서 재생성 버튼을 숨기는 기존 수정은 유지했다.

## 검증 결과

```powershell
npm.cmd --prefix admin-web run typecheck
```

결과: 성공

브라우저 확인:

- URL: `http://localhost:5173/ai-assets`
- 반려 산출물 `#31102` 상세에서 재생성 요청 모달 열림 확인
- 이미 active job이 있는 상태에서 재생성 요청 제출
- 모달이 닫히고 `이미 재생성 작업이 진행 중입니다.` 안내가 표시됨
- 상세 Drawer 상단이 `재생성` 버튼에서 `재생성 작업 진행 중 (QUEUED/RUNNING)` 표시로 변경됨

## 결론

첫 재생성 요청은 정상 접수됐는데 같은 버튼을 다시 누를 수 있어서 실패처럼 보이는 문제가 있었다. 서버 계약은 유지하고, 프론트가 현재 세션에서 확인한 재생성 진행 상태를 표시하도록 바꿔 사용자가 중복 요청을 실패로 오해하지 않게 했다.

## 후속 메모

페이지 새로고침 후에도 active job 표시가 필요하면 상세 API에 active generation job 요약 필드를 추가해야 한다. 이번 작업은 서버 변경 없이 현재 관리자 화면의 즉시 혼란을 줄이는 범위로 제한했다.
