## TODO 1. [P2] 공개 경로 rate limit — service-user

**문제**: `POST /api/v1/auth/kakao`·`/api/v1/auth/refresh`·`/api/v1/admin/auth/kakao`가 무제한 호출 가능. 카카오 토큰 무차별 대입·refresh 남용을 막을 1차 방어가 없다.

**해결책**: 이미 쓰는 Redis(`RefreshTokenStore`)로 IP+경로 단위 고정창 카운터 필터. 별도 라이브러리 없이 시작 가능(필요해지면 bucket4j 검토 — 금지기술 아님).

```java
// security/RateLimitFilter.java (service-user, OncePerRequestFilter)
// 키: rl:{path}:{clientIp}:{epochMinute} — INCR 후 첫 증가면 EXPIRE 60s
// 한도 예: /auth/kakao 10회/분, /auth/refresh 30회/분 (yml로 주입)
Long count = redis.opsForValue().increment(key);
if (count != null && count == 1L) redis.expire(key, Duration.ofSeconds(60));
if (count != null && count > limit) {
    response.setStatus(429);  // 공통 에러 봉투(ErrorCode.TOO_MANY_REQUESTS 신설)로 작성
    return;
}
```

- [ ] 대상 경로·한도를 yml 프로퍼티로(`security.rate-limit.…`), 기본은 보수적으로
- [ ] X-Forwarded-For 처리: gateway(nginx, 강태오 작업) 뒤에 서게 되므로 신뢰 가능한 첫 IP 사용 — gateway PR과 순서 조율
- [ ] 429 응답도 공통 `ApiResponse` 에러 형식 유지, **로그에 토큰·개인정보 금지**(CLAUDE.md §9)
- [ ] 테스트: 한도 내 통과/초과 429/창 리셋(분 경계) — Redis는 기존 테스트 구성(또는 embedded) 활용
- [ ] Redis 장애 시 fail-open(로그인 가용성 우선) 정책 명시 — try-catch로 카운터 실패 시 통과 + warn 로그

**검증**: `:service-user:test` + 로컬에서 curl 반복 호출로 429 확인.

## TODO 2. [P2] flutter 웹 카카오 로그인 사전 가드

**문제**: 카카오 dart SDK는 웹에서 동작하지 않는데 로그인 화면에 `kIsWeb` 분기가 없어, 웹 사용자가 버튼을 누르면 SDK 예외가 그대로 발생. (TTS는 `kIsWeb` 분기가 잘 돼 있는 것과 대조적)

**해결책**: 로그인 버튼 진입부에서 웹이면 비활성화 + 안내.

```dart
// 로그인 화면 빌드부
final webLoginUnsupported = kIsWeb && !WebDevAccess.enabled; // 기존 삼중 게이트 유틸 재사용
// 버튼: onPressed: webLoginUnsupported ? null : _onKakaoLogin
// 안내: l10n 키 loginWebNotSupported = "웹에서는 카카오 로그인을 지원하지 않습니다. 모바일 앱을 이용해 주세요."
```

- [ ] l10n 키 추가(ko/en) — 하드코딩 금지
- [ ] `WebDevAccess`(dev 전용 우회) 경로는 기존 동작 유지
- [ ] 위젯 테스트 1건: kIsWeb 시 버튼 비활성+안내 노출 (`debugDefaultTargetPlatformOverride`가 아닌 kIsWeb은 테스트 분기 어려우므로, 가드 로직을 순수 함수로 분리해 단위 테스트)
- [ ] 참고: 웹 카카오 로그인 정식 지원은 서버측 OAuth가 필요해 기존 결정과 충돌하는 미해결 설계 이슈 — 이번 작업은 "안내 가드"까지만

**검증**: `flutter analyze` + 테스트, 웹 dev 실행으로 안내 표시 확인.

## TODO 4. [P3] 카카오 네이티브 키 기본값 제거

**문제**: `app_config.dart:42`에 dev 네이티브 앱 키가 기본값으로 커밋돼 있음. APK에 포함되는 공개 식별자라 실위험은 낮지만(주석으로 사유 명시됨), CLAUDE.md §8 "plain secret 예시 금지"를 엄격 해석하면 빈 기본값+주입 강제가 깔끔하다.

- [ ] `defaultValue: '53e5…'` → `''`
- [ ] 실행 가이드(README/run-dev-web.ps1 안내)에 `--dart-define=KAKAO_NATIVE_APP_KEY=...` 명시
- [ ] 팀 공지: 로컬 실행 시 키 주입 필요해짐(빈 키면 카카오 로그인만 비활성, 나머지 개발 가능)

**검증**: 키 없이 dev 실행 → 경고 로그+앱 정상 동작, 키 주입 시 로그인 동작.
