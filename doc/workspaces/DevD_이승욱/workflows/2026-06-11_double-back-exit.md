# 2026-06-11 뒤로가기 2번으로 앱 종료 (feature/mobile-double-back-exit)

## 목표·배경
홈(루트)에서 뒤로가기 한 번에 앱이 바로 종료되는 문제 — 실수 종료 방지를 위해 안드로이드 관례(첫 입력 안내 → 2초 내 재입력 시 종료)로 변경 요청.

## 작업 내용
- `DoubleBackExitPolicy`(home/services): 종료 판정 순수 함수 — 창 2초, 경계 포함. 시간 판정을 분리해 위젯 없이 단위 테스트.
- `HomeScreen`: `PopScope(canPop: false, onPopInvokedWithResult: ...)` — 첫 입력은 플로팅 스낵바 안내(노출 시간 = 판정 창 2초로 일치), 창 내 재입력은 `SystemNavigator.pop()`으로 태스크 종료. 홈 위에 push된 상세 화면들은 자기 라우트가 pop을 처리하므로 영향 없음.
- l10n `homeBackExitGuide`(ko/en) — 하드코딩 없음.

## 추가 보강 (T 요청: 하위 화면 복귀 보장)
- 요구 확인: ① 다섯 탭 어디서든 뒤로가기 2번 = 종료 ② push된 하위 화면(마이→알림/설정, 노트→작성/상세 등)의 뒤로가기 = 부모 화면 복귀.
- 점검 결과 ②는 라우트 스택의 기본 pop으로 이미 동작(스택을 끊는 pushReplacement류는 로그인/로그아웃/탈퇴 흐름뿐임을 전수 grep으로 확인). 다만 보장이 없어:
- 종료 가드를 `DoubleBackExitScope` 위젯으로 추출(홈은 Dio 호출 화면 포함이라 직접 위젯 테스트 불가 — 기존 widget_test 주석의 제약) 후, **실제 시스템 뒤로가기(`handlePopRoute`) 위젯 테스트 3건**으로 ①②를 고정:
  - push된 화면 뒤로가기 → 부모 복귀 + 가드(스낵바/종료) 미개입
  - 루트 첫 입력 → 스낵바만, 종료 없음
  - 2초 내 재입력 → `SystemNavigator.pop` 호출(플랫폼 채널 캡처로 검증)

## 검증
- 정책 단위 3건 + 스코프 위젯 3건, `flutter analyze` 무이슈, `flutter test` 162건 전체 통과.
- 수동: 에뮬레이터에서 홈 뒤로가기 1회(안내)·2회(종료), 마이→알림/설정·노트→작성 뒤로가기 복귀 확인 예정.
