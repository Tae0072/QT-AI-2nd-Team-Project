## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| 묵상 범위 | 묵상은 오늘 QT 본문 전용이다. 자유 본문용 수동 생성 API는 만들지 않는다. |
| 묵상 생성 | `GET /api/v1/notes/draft`와 `POST /api/v1/notes` 기준으로 DRAFT를 조회·생성한다. |
| 저장 필드 | 느낀 점, 기억할 구절, 적용할 점, 기도 4개 섹션을 분리한다. |
| 글자 수 | 사용자 입력 글자 수 제한은 두지 않는다. 악성 요청 방어는 서버 공통 정책으로 처리한다. |
| 이벤트 | v1은 Spring `ApplicationEventPublisher` 기반 인프로세스 이벤트다. |
| 찬양 | AI 추천이 아니라 운영자가 사전 큐레이션한 곡 목록에서 저장/제거한다. 가사·음원 파일은 저장하지 않는다. |
| 도메인 패키지 | `domain.note`/`domain.sharing`/`domain.praise`는 모두 `api/internal/client/web` 5층 구조. `internal/`은 외부 접근 절대 금지. 다른 도메인은 `note.api.*UseCase`만 호출 가능. 다른 도메인 호출 시 Mock은 `client/{타도메인}/...UseCaseMock.java`(03 v1.2 §3.1). |
| 노트 `@`멘션 동기화 | 클라이언트가 본문에 인용 블록 삽입 시 `verseIds` 배열을 함께 전송. 서버는 `body`를 자유 형식으로 보존하고 `verseIds`를 `note_verses`에 중복 없이 저장. 동일 절을 두 번 인용해도 `note_verses`에는 한 번만 기록(04 v1.4 §4.3.6). |
| 닉네임 변경 영향 | 노트 작성자 닉네임이 변경되어도 기존 `sharing_posts.nickname_snapshot`은 자동 변경하지 않는다. 공유 시점 스냅샷 정책 그대로(07 v3.3 §F-10, 04 v1.5 §4.1.5). |
| 노트 저장 정책 (v1) | 현행 서버 마스터 단일 — 노트 도메인은 **현행 그대로**. F-03/F-13/F-04/F-10 정책 변경 없음. v1.1 이후 (b) 절충안(서버 마스터 + 로컬 캐시 + 작성 시 큐잉) 사전 인지하되 v1 본문 변경 없음(07 §18.2). |

## 5. 브랜치·PR·워크플로우 기록 규칙

| 항목 | 이승욱 기준 |
| --- | --- |
| 브랜치 생성 | `dev` 최신화 후 `feature/note-*`, `feature/praise-*`, `feature/sharing-*` 형식으로 생성 |
| PR 대상 | 항상 `dev` |
| 자동 PR | 브랜치 push 후 GitHub PR을 생성하면 CI와 자동 리뷰가 실행된다. 자동 머지는 하지 않는다. |
| 작업 폴더 | 별도 구현 GitHub에서 `workspaces/이승욱/workflows`, `workspaces/이승욱/reports`를 사용 |
| 기록 순서 | 작업 전 workflow 작성 → 작업·검증 → report 작성 → PR 생성 |
| 기준 문서 | 상세 명령과 템플릿은 `개발자별_일정표/00_공통_브랜치_PR_워크플로우_규칙.md`를 따른다. |

## 6. PR 전 체크

| 체크 | 기준 |
| --- | --- |
| Note 서비스 | 독립 서비스로 분리하지 않고 `domain.note` 모듈로 유지 |
| 패키지 구조 | `api/internal/client/web` 5층 준수. 다른 도메인 호출은 `client/{도메인}/...UseCaseMock` 사용 |
| 자유 본문 | 자유 본문 묵상 생성 API 없음 |
| AI 연결 | 묵상 노트가 AI 세션과 연결되지 않음 |
| `@`멘션 동기화 | 노트 저장/수정 시 `verseIds`로 `note_verses` 메타데이터 동기화. `body`는 자유 형식 보존 |
| 닉네임 스냅샷 | 닉네임 변경 시 `sharing_posts.nickname_snapshot` 자동 변경 안 함 |
| 이벤트 | 핸들러 실패 로그와 재처리 가능 기준을 남김 |
| 찬양 | AI 추천, 직접 URL 입력, 가사·음원 저장 없음 |
| v1.1 자안 사전 인지 | 노트 로컬 캐시·큐잉(b) 자안은 v1에서는 구현하지 않음. v1.1 진입 시 PR 묶음으로 처리 |
