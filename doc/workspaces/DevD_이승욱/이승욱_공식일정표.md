## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| 묵상 범위 | 묵상은 오늘 QT 본문 전용이다. 자유 본문용 수동 생성 API는 만들지 않는다. |
| 묵상 생성 | `GET /api/v1/notes/draft`와 `POST /api/v1/notes` 기준으로 DRAFT를 조회·생성한다. |
| 저장 필드 | 느낀 점, 기억할 구절, 적용할 점, 기도 4개 섹션을 분리한다. |
| 글자 수 | 사용자 입력 글자 수 제한은 두지 않는다. 악성 요청 방어는 서버 공통 정책으로 처리한다. |
| 이벤트 | v1은 Spring `ApplicationEventPublisher` 기반 인프로세스 이벤트다. |
| 찬양 | AI 추천이 아니라 운영자가 사전 큐레이션한 곡 목록에서 저장/제거한다. 가사·음원 파일은 저장하지 않는다. |

## 5. 브랜치·PR·워크플로우 기록 규칙

| 항목 | 이승욱 기준 |
| --- | --- |
| 브랜치 생성 | `dev` 최신화 후 `feature/note-*`, `feature/praise-*`, `feature/sharing-*` 형식으로 생성 |
| PR 대상 | 항상 `dev` |
| 자동 PR | 브랜치 push 후 GitHub PR을 생성하면 CI와 자동 리뷰가 실행된다. 자동 머지는 하지 않는다. |
| 작업 폴더 | 별도 구현 GitHub에서 `workspaces/이승욱/workflows`, `workspaces/이승욱/reports`를 사용 |
| 기록 순서 | 작업 전 workflow 작성 → 작업·검증 → report 작성 → PR 생성 |
| 기준 문서 | 상세 명령과 템플릿은 `개발자별_일정표/00_공통_브랜치_PR_워크플로우_규칙.md`를 따른다. |

## 6. PR 전 체크

| 체크 | 기준 |
| --- | --- |
| Note 서비스 | 독립 서비스로 분리하지 않고 `domain.note` 모듈로 유지 |
| 자유 본문 | 자유 본문 묵상 생성 API 없음 |
| AI 연결 | 묵상 노트가 AI 세션과 연결되지 않음 |
| 이벤트 | 핸들러 실패 로그와 재처리 가능 기준을 남김 |
| 찬양 | AI 추천, 직접 URL 입력, 가사·음원 저장 없음 |
