### W2 시작 전 합의·확인 (5/22 금까지)

> 회의록 §3 분담 기준으로 의존성 정리. **김지민이 만들 코드는 다른 팀원의 산출물 위에서 동작**하므로, W2 시작 전 다음 항목들의 진행 상황을 반드시 확인.

**이지윤 (bible · 묵상·설교 노트 백엔드) 진행 상황**

- [ ] **Note Entity 시그니처 합의** (이지윤 소유) — `qtId` nullable + `category` enum(PRAYER/REPENTANCE/GRATITUDE/MEDITATION/SERMON) + `title` 필드. `notes` 테이블을 자유 노트와 공유.
- [ ] **노트 목록 조회 API 의존성** (이지윤 소유, **김지민 X**) — `GET /api/v1/notes` (카테고리 필터). N-01 화면이 5개 카테고리 통합 조회. 김지민은 Flutter에서 호출만.
- [ ] **묵상 달력 API 의존성** (이지윤 소유, **김지민 X**) — `GET /api/v1/me/meditation-calendar`. 5개 카테고리 통합 응답. 김지민은 응답 형식·필드를 적극 요구할 수 있음 (화면 책임자라).

**이승욱 (인증·회원 백엔드 + 팀 공통 기반) 진행 상황**

- [ ] **공통 응답 Envelope** (이승욱 소유) — `ApiResponse { success, data, error, timestamp, traceId }`. 김지민의 모든 Controller가 이걸로 응답 포장.
- [ ] **공통 에러 코드** (이승욱 소유) — `DUPLICATE_NOTE`, `TOKEN_EXPIRED`, `FORBIDDEN` 등. Service에서 던질 예외 코드.
- [ ] **나눔 Entity 4종** (이승욱 소유) — `SharingPost`, `Comment`, `PostLike`, `Report`. W2 `GET /sharing-posts`·W3 좋아요·댓글·신고에서 모두 사용.
- [ ] **JWT 인증 & Security Filter** (이승욱 소유) — `@AuthenticationPrincipal` 로 `memberId` 받기. 김지민의 모든 API가 이걸로 인증.
- [ ] **Flutter 프로젝트 초기 설정** (이승욱 소유, **김지민 X**) — 상태관리·HTTP 통신·라우팅·공통 상태 위젯·에러 인터셉터까지. 김지민이 `flutter create` 하는 게 아니라 받아 쓰는 입장. 머지 후 김지민 화면 작업 시작 가능.

**김지민 자체 합의 사항**

- [ ] **자유 노트 엔드포인트 합의** — `POST /api/v1/notes` (qtId 없음, 자유 노트) vs 기존 골격의 `POST /api/v1/notes/qt/{qtId}` (묵상 노트, 이지윤 소유) 경로 충돌 정리. 둘 다 공존할지, 통합할지 이지윤과 합의.
