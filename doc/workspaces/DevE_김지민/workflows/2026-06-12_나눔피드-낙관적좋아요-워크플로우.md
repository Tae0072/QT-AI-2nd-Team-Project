## 배경 / 결정

- **카드 레이아웃은 이미 적용돼 있었다**(`PostCard` 위젯 = Calm Paper 카드). 그래서 ③의 실제 작업은
  **좋아요 낙관적 업데이트**다.
- 기존 피드의 하트는 **표시 전용**(탭 불가)이었고, 좋아요 토글은 상세(S-02)에서 `invalidate` →
  **전체 재조회·재렌더(깜빡임)** 였다.

## 변경 내용

- `sharing_providers`: `sharingPostsProvider`를 **`FutureProvider` → `AsyncNotifierProvider`**(`SharingFeedNotifier`)로
  전환. `toggleLike(postId)`를 추가 — **즉시 로컬 상태 갱신(낙관적)** 후 서버 반영, **실패 시 롤백**.
  invalidate/refetch 없이 해당 글의 `likeCount`/`likedByMe`만 바뀐다.
- `SharingPostItem.copyWith(likeCount, likedByMe)` 추가(낙관적 갱신용).
- `PostCard`: 하트를 **탭 가능**하게(`onLike` 콜백). 카드 탭(상세 이동)과 분리(InkWell이 자기 영역 탭을 가져감).
- `sharing_feed_screen`: `PostCard.onLike = notifier.toggleLike(id)` 배선 + 실패 시 안내 스낵바(`sharingLikeFailed`/
  `sharingUnlikeFailed`).
- `ref.watch`/`ref.invalidate`(RefreshIndicator·상세·M-05) 인터페이스는 AsyncValue 호환이라 그대로 동작.

## 상세(S-02)도 낙관적 전환 (같은 PR에 포함)

피드만 고치면, 사용자가 실제로 하트를 누르는 **상세 화면**은 여전히 `invalidate`로
본문+댓글까지 재조회·깜빡였다. 그래서 상세도 같은 방식으로 전환했다.

- `sharingPostDetailProvider`를 **`FutureProvider.family` → `AsyncNotifierProvider.autoDispose.family`**
  (`SharingDetailNotifier`)로 전환. `toggleLike()` 추가 — detail의 좋아요 수/하트만 즉시 바꾸고
  **댓글 목록은 그대로 유지**(재조회 없음), 실패 시 롤백.
- `SharingPostDetail.copyWith(likeCount, likedByMe)` 추가.
- `sharing_detail_screen._toggleLike`: 기존 `repo.like/unlike + _refresh()`(전체 재조회) →
  `notifier.toggleLike()`(낙관적)로 교체. 좋아요 성공 후 **피드는 백그라운드로 `invalidate`**
  (상세 화면은 안 깜빡이고, 돌아갔을 때 피드 좋아요 수가 어긋나지 않게). 댓글 작성·삭제·신고·글 삭제는
  기존 `_refresh()`/`invalidate` 유지(거긴 재조회가 맞다).

## 남은 리스크 / 후속
- 낙관적 갱신은 화면이 유지되는 동안만(autoDispose). 화면 재진입 시 서버 기준 재조회.
- 상세 좋아요 후 피드 `invalidate`는 기존 동작과 동일(피드는 상세 뒤에 있어 사용자에게 깜빡임이 보이지 않음).
