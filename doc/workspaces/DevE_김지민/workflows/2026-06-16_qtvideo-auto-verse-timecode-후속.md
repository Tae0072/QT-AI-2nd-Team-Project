## 배경

QT 클립의 시작/끝 초는 "각 절이 영상 몇 초인지"(절↔초 매핑)에서 나온다.

- QT 범위 → 클립 start/end **자동 계산은 이미 동작**한다(`prepareClip`이 절별 구간의 `min~max`를 잡음).
- 빠진 것은 **절별 타임코드 자체를 자동 생성**하는 단계다. 현재는 사람이 `bible_verse_video_segments`를 손으로 입력해야 한다.

## 본 PR에서 한 것(하이브리드 1단계)

- **직접 클립 생성**(`POST /api/v1/admin/qt-videos/clips/manual`): 절별 구간 없이 `qtPassageId + sourceVideoId + startTimeSec + endTimeSec`만으로 활성 클립을 생성·교체.
- 절별 구간 방식(`replaceSegments` → `prepareClip`)도 그대로 유지.
- → 한 본문을 빠르게 맞출 땐 "직접 클립", 책 전체를 한 번 찍어 재사용할 땐 "절별 구간"을 선택 가능.

## 후속 구현(2단계): 절↔초 자동 매핑

### 목표
원본 영상이 담는 절 범위를 기준으로 `bible_verse_video_segments`를 **자동 생성**해, 관리자가 타임코드를 손으로 찍지 않게 한다. 클립/유저API/플러터는 무수정(절별 구간만 채워지면 기존 흐름이 그대로 동작).

### 방법 후보
| 방법 | 입력 | 정확도 | 비용 |
|---|---|---|---|
| A. 균등 분할 | 영상=책/장 전체, 0~길이초 | 낮음 | 매우 낮음 |
| **B. 글자수 가중 분할(권장)** | 위 + `bible_verses` 본문 길이 | 중간 | 낮음 |
| C. 강제 정렬(forced alignment) | 영상 음성 STT + 절 텍스트 정렬 | 높음 | 높음(ML 파이프라인) |

### B 구현 시 수정 지점
1. **원본 영상 커버 범위**: source_videos에 "이 영상이 담는 절 범위 + 선형 매핑" 가정(또는 `source_video_chapter_anchors` 같은 장별 시작초 보조 입력).
2. **자동 구간 생성 서비스**(admin-server `qtvideo`): bible.api로 책 절 목록(텍스트 포함)을 순서대로 조회 → 글자수 비율로 초 배분 → `bible_verse_video_segments` 저장.
   - bible.api 보강 필요 가능: "책 전체 절 목록(텍스트 포함)" 조회.
3. **엔드포인트 + admin-web 버튼**: `POST /admin/qt-videos/source-videos/{id}/segments/auto-generate` + "구간 자동 생성".
4. 클립 생성/유저 노출/플러터: **수정 없음**.
5. 테스트: 글자수 가중 배분 단위 테스트 + `@DataJpaTest`.

### 한계
A/B는 근사치(내레이션 속도·인트로·묵음으로 오차). 정확히는 C(강제 정렬) 필요.
워크플로우 권장: **자동 생성으로 초안 → 경계 절(첫·끝 절)만 사람이 미세 조정**(클립 컷은 min/max만 쓰므로 첫·끝 절만 정확하면 충분).
