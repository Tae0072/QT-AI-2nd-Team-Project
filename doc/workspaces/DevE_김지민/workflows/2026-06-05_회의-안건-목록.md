## 2. 묵상(QT) 노트 작성·수정 화면
- **배경**: 묵상 노트 CRUD는 **백엔드 100% 완비**(서버 PATCH가 4섹션 처리). 필요한 건 백엔드가 아니라 **Flutter 화면**.
- **진행(2026-06-08)**: 이지윤 PR로 **작성 화면 `QtNoteEditorScreen` 생김** + `today_qt_screen` "노트" 버튼 연결됨(**빈 껍데기 해소**). 단 그 에디터는 `passage`만 받는 **작성 전용**(noteId 없음 = 수정 모드 없음).
- **남은 것**: QT 노트 **수정 화면(수정 모드)** 부재 → N-04에서 묵상 [수정]은 여전히 숨김.
- **분담 제안** (편집 화면 = 이지윤 / N-04 진입 = 김지민):
  - **이지윤**: `QtNoteEditorScreen`에 **수정 모드** 추가 — noteId args → GET으로 4섹션 prefill → 저장 시 **PATCH /notes/{id}**. (서버는 이미 완비, **Flutter 화면만**)
  - **김지민**: N-04에서 묵상 [수정] 버튼 다시 켜고(`writableNoteCategories` 제외 해제) → QT 에디터 수정모드로 **라우팅**(작은 작업).
  - **공유 파일 보완**: Flutter `note_repository.update()`가 현재 4섹션을 안 보냄(title/body만) → 4섹션 전송 추가 필요(누가 할지만 합의).
  - **진입점 2개 → 화면 1개**: QT 노트 수정은 ⑴ today_qt "노트" 버튼(오늘 노트 있으면 수정), ⑵ N-04 [수정] 버튼 두 곳에서 같은 에디터를 열게 됨. 그래서 `QtNoteEditorArgs`가 **passage(작성/오늘)·noteId(수정/특정 노트) 둘 다** 받게 설계하면 화면 하나로 두 진입 모두 처리.
  - **순서**: 이지윤 수정모드 + args 계약 **선행** → 김지민 연결.
- **결정 필요(가벼움)**: ① QT 수정 화면 담당이 이지윤 맞는지 확인 ② N-04 → 에디터 **args 계약**(noteId 넘기는 방식) 한 줄. 본격 설계 회의는 불필요.

## 3. 노트 본문 서식 툴바 — "마커가 사용자에게 그대로 노출" 문제
- **배경**: 서식 툴바(A 방식)가 Bold 누르면 `**` 같은 **마크다운 마커를 글자로 삽입**(`body`가 평문이라). 사용자에겐 `**굵게**`가 그대로 보여 "컴퓨터만 이해하는" 형태.
- **2026-06-08 검토 — 세 가지 선택지**:

  | 옵션 | 작업량 | 합의 | 결과 |
  |---|---|---|---|
  | **없애기**(툴바 제거) | ~30분(1~2파일) | 제품결정(Lead 확인) | 서식 기능 사라짐, 마커 안 생김. §19.1 계획 기능 제거 + 이지윤 QT툴바와 불일치 |
  | **미리보기**(저장 평문 유지) | ~반나절 | 불필요(솔로) | 입력칸 아래 렌더 미리보기. 기능 유지, 마커 불편 완화 |
  | **리치텍스트**(Delta/HTML) | 며칠+ | 이지윤·백엔드 필수 | 시각적 서식, 마커 없음. 큰 공동작업 |

- **리치텍스트가 "혼자 결정 불가"인 이유 — `body`가 공유 계약**: `notes.body`를 평문→Delta/HTML로 바꾸면 그 포맷이 흐르는 **4층 전부**를 같이 고쳐야 함.
  - ① 저장: `notes.body`(+포맷 마커 컬럼 가능 → 02 ERD), 공유 시 `sharing_posts.body_snapshot` 복사(`composeBody`)
  - ② 입력: 김지민 자유노트 툴바/N-03 + **이지윤 QT 툴바/QT 에디터**(같은 포맷으로 맞춰야 함)
  - ③ 표시: N-04 상세·나눔 상세·나눔 피드 미리보기(평문 Text → 리치 렌더)
  - ④ 외부 공유: 카톡 전송·카드 이미지(리치를 평문으로 flatten 안 하면 Delta JSON 노출)
  - ⚠️ **데이터 혼재**: 기존 평문 + 신규 Delta가 같은 컬럼에 섞임 → 포맷 버전 마커 또는 마이그레이션 필요.
  - **담당**: 자유노트 입력·표시·공유=김지민 / QT 툴바·에디터=이지윤 / body_snapshot·(마커 컬럼)ERD=백엔드 → **3명·2도메인 + 마이그레이션**.
- **결정 필요**: 위 3옵션 중 택1. (없애기=Lead 확인 / 미리보기=김지민 솔로 / 리치텍스트=공동·일정 확보). **이번 달력 브랜치에서는 툴바 코드 안 건드림 — 회의 결정 후 별도 진행.**

## 4. 설교노트 편집 화면(B-03)
- **배경**: N-04 단일본문 편집(N-03 재사용)은 `verseIds`를 안 보냄 → 설교노트를 PATCH하면 **인용 절(note_verses)이 삭제됨**(04 §4.3.6). 그래서 오늘 N-04 [수정]을 **자유노트(기도/회개/감사)에만** 노출하고 설교노트는 제외.
- **핵심**: verseIds를 update에 끼워넣는 건 해결책이 아님 — N-03엔 **절을 고르는 UI가 없어** 유지만 가능하고 편집 불가. **절 선택 UI가 있는 화면(B-03)**이 본질적 해결.
- **분담 제안** (편집 화면 = 이지윤 / N-04 진입 = 김지민):
  - **이지윤**: **B-03 절 선택 편집 화면** — verseIds를 제대로 PATCH. (bible 영역, 거의 합의 불필요)
  - **김지민**: N-04에서 설교노트 [수정] 버튼 → B-03 **연결**.
  - **순서**: 이지윤 B-03 선행 → 김지민 연결.
- **결정 필요**: B-03은 명백히 이지윤 영역 → 연결 **시점**만 맞추면 됨.
