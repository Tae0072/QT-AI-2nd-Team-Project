## 1. 배경 — 왜 만들어졌나

sharing 도메인 안에 **이름이 비슷한 두 시스템**이 공존한다.

| 구분 | 정체 | 상태 | 04 명세 |
|---|---|---|---|
| `sharing-posts` | 닉네임 나눔 게시글 (앱 내 피드, 좋아요·댓글) | ✅ 완성 | 있음 (`/sharing-posts/**`, `POST /notes/{id}/share`) |
| `shares` | **토큰 링크 공유** (`/api/v1/shares`, 비로그인 스냅샷 조회) | ❌ TODO 껍데기 | **없음** |

`shares`는 "앱 미설치자가 공유 링크(`GET /shares/token/{token}`)를 눌러 로그인 없이 스냅샷을 보는" 흐름으로 설계됐다(스텁 주석 기준). 이를 렌더링할 **공개 웹페이지가 전제**인데, 현재 프로젝트엔 admin 웹 외 공개 웹이 없다.

## 2. 죽은 코드 판정 근거 (2026-06-05 grep 전수 검증)

1. **소비처 0** — `shares` UseCase를 쓰는 곳은 `report/client/sharing/GetSharedSnapshotUseCaseMock.java` 단 하나인데, 그것도 `// TODO: implements`인 또 다른 껍데기.
2. **report 실제 코드는 안 씀** — `ReportService.java:8`은 `GetSharedSnapshotUseCase`가 아니라 **`GetSharingPostUseCase`(완성된 sharing-posts)**를 import. 신고 검증도 실제론 나눔게시글을 사용.
3. **Flutter 미호출** — 앱은 `POST /notes/{noteId}/share`(나눔게시글 생성)만 호출. `/api/v1/shares`·`/token/`은 어디서도 호출 안 함.
4. **SecurityConfig·테스트 영향 없음** — `/shares`·`/token` permitAll 설정 없음(TODO 주석만), 테스트 참조 없음.
5. **외부 참조는 주석 1줄뿐** — `SharingPostController.java:39` Javadoc이 SharingController를 언급(컴파일 무관).

→ **04 명세에도 없고 + 공개 웹도 없고 + 코드 소비처도 없는 3중 죽은 코드.**

## 3. ⚠️ 혼동 주의 — "외부 공유" 기능은 살아있다

07 요구사항 §19.1에 **김지민 담당 "N-04 외부 공유 버튼 / 텍스트·카드이미지(share_plus, RenderRepaintBoundary)"는 v1 포함**이다. 단 이건 `shares`와 **완전히 다른 것**:

| | `shares` 토큰 링크 (죽음) | §19.1 외부 공유 (살아있음) |
|---|---|---|
| 방식 | 서버가 스냅샷 저장 + 공개 URL 발급 | Flutter `share_plus` = OS 공유시트로 텍스트/이미지 전송 |
| 서버 | 필요(현재 껍데기) | **불필요(100% 클라이언트)** |

**`shares` 폴더를 지워도 외부 공유 기능은 멀쩡하다.**
