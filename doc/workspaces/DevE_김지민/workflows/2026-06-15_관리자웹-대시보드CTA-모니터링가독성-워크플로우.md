## 1. 문제 정의

### 문제 1 — AD-01 대시보드 조치 동선 단절 (핵심 결함)
대시보드(`DashboardPage.tsx`)가 운영 지표·오늘 QT 문제 상태를 **표시만** 하고, 문제에서 바로 처리 화면으로 가는 동선이 없다. 운영자는 좌측 메뉴로 직접 화면을 찾아 맥락을 다시 잡아야 한다 → 대시보드의 "한눈에 보고 즉시 조치" 목적이 깨짐.

### 문제 2 — AD-08 모니터링 가독성·노출 범위 (개선)
`AiMonitoringPage.tsx`는 집계 카드·코드별 테이블·전체로그 CTA·"원문 미포함" 정책을 이미 갖췄으나, 데이터가 차면:
- 실패/차단 사유 테이블이 개수 제한·정렬 없이 전부 노출되어 길어짐.
- 배치 `errorMessage`를 원문 그대로 렌더해 행이 깨지고 내부 정보가 과다 노출됨.

## 2. 해결 방안 (완벽한 구현 조건)

### Part 1 — AD-01 대시보드 CTA (구현, **A1: 정밀 딥링크**)
`DashboardPage.tsx`에 `useNavigate` 도입 후, 문제 상태에만 조치 CTA 노출.
"정밀 딥링크(A1)" = 도착 화면이 존재하는 동선은 `?status=`/`?focusId=` 쿼리로 **정확한 맥락**에 도착시킴. 도착 화면이 없는 동선(시뮬레이터 생성)은 가짜 CTA를 만들지 않고 후속 PR로 분리.

| 위치 | 노출 조건 | CTA 대상 | 도착 정합성(A1) |
|---|---|---|---|
| "AI 검증 대기" 카드 | `pendingAiValidationCount > 0` | `/ai-assets` | AD-03 기본필터 VALIDATING → 정밀 도착(대상 화면 변경 불필요) ✅ |
| "신고 접수" 카드 | `receivedReportCount > 0` | `/reports?status=RECEIVED` | ReportsPage가 `?status=` 초기필터 수용(보강) → 정밀 ✅ |
| "신고 검토 중" 카드 | `reviewingReportCount > 0` | `/reports?status=REVIEWING` | 〃 ✅ |
| 오늘 QT · 상태 | `status ≠ READY` | `/qt-passages?focusId={qtPassageId}` | QtPassagesPage가 `?focusId=` 수용 → 그 본문 강조/열기 → 정밀 ✅ |
| 오늘 QT · 해설 | `hasExplanation = false` | `/ai-assets` | 화면은 정확(AI 산출물). per-passage 정밀필터(verse 단위)는 후속 — 현재 generic 도착 |
| 오늘 QT · 시뮬레이터 | `simulatorStatus ≠ READY` | (임시) `/qt-passages?focusId={qtPassageId}` | **전용 관리 화면 부재(B-1)** → 임시로 본문 관리로 이동 + 후속 PR로 전용 화면 신설 |

**완벽 조건(A1)**
- 정상(0건 / READY / 해설 있음)이면 CTA 숨김 — 노이즈 제거.
- 클릭 affordance 명확(링크/버튼 + 문제 색 강조).
- `qtPassageId`가 `null`이면(오늘 QT 미등록) 오늘 QT 관련 CTA는 `/qt-passages`로만 이동(focus 없음).
- 시뮬레이터 동선은 가짜 정밀화 대신 임시 이동 + 후속 명시(§6).

### Part 2 — AD-08 모니터링 가독성 (개선)
`AiMonitoringPage.tsx`:
1. 검증 실패 사유 / Q&A 차단 사유: `count` 내림차순 + 상위 5개만, 나머지는 "그 외 N개" 요약 행.
2. 배치 실패 `errorMessage`: `ellipsis` + `Tooltip`(hover 전체) 처리.
3. "원문 미포함" 정책 유지.

### 범위 경계 (A1)
- admin-web FE만 수정. BE/스키마/타 도메인 변경 없음.
- ReportsPage·QtPassagesPage 보강은 **URL 쿼리 초기필터/포커스 수용**(작은 additive 변경)에 한정. 처리/CRUD 로직 불변.
- 시뮬레이터/QT영상 전용 관리 화면 신설은 본 PR 범위 밖 → **후속 PR(§6)**.
