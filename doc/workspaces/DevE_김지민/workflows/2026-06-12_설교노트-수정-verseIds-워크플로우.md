## 배경

- **발견(⑫):** N-04 상세의 [수정] 버튼이 **자유노트(기도/회개/감사)에만** 노출되고 **QT·설교는 수정 불가**였다. 코드 주석에 차단 사유가 명시돼 있었다 — "설교노트를 PATCH하면 verseIds 미전송으로 note_verses가 비워짐(04 §4.3.6)".
- **연결(⑪):** 그 차단 사유가 곧 ⑪의 작업 대상 — `update`에 verseIds를 실어 보내면 절 참조가 보존돼 설교 노트 수정이 가능해진다. 그래서 ⑪과 설교 수정(⑫ 일부)을 한 PR로 묶었다.
- **QT(MEDITATION)는 이번 범위 제외:** QT 노트는 4섹션 데드코드(`rememberSection` 등을 쓰는 화면 없음, QT 에디터는 단일 body 저장)와 N-04 표시 불일치(빈 4섹션 표시) 정리가 선행돼야 함 → **Lead 확인 후 별도**.

---

## 변경 내용

### 1. `note_repository.dart`
- `create`/`update`에 `List<int> verseIds = const []` 추가 → POST/PATCH body에 `verseIds` 포함(04 §4.3.4·§4.3.6). `update`는 전달한 배열로 `note_verses`를 교체하므로, 수정 시 기존 절을 그대로 다시 보내 보존한다.

### 2. `note_edit_screen.dart` (N-03)
- `NoteEditArgs`에 `verseIds`(List<int>?) 추가 — 작성 진입 시 인용 절 동봉(② 설교 버튼이 `NoteEditArgs(category:'SERMON', verseIds:[...])`로 넘기면 바로 연결).
- `_verseIds`(Set<int>)에 모은다: 작성=`args.verseIds` 시드 / 편집=`detail.verses` 시드 / + @멘션 삽입분 누적(중복 제거). 저장 시 `create`/`update`에 전달.

### 3. `note_rich_text_editor.dart` (공유 에디터)
- `onVerseInserted(List<int> verseIds)` 콜백 추가(선택). @멘션 구절 삽입 시 그 절들의 `bibleVerseId`를 부모에 알린다. **QT 에디터는 콜백 미사용(영향 0)**, N-03만 사용.

### 4. `note_detail_screen.dart` (N-04)
- 수정 버튼 게이트를 `_isEditable(category)` = 자유노트 + **SERMON**으로 확장(QT 제외). 설교 노트는 ⑪의 verseIds 보존 PATCH 덕분에 손상 없이 수정 가능.

## 남은 리스크 / 후속
- **② 설교 작성 화면(이지윤)** 이 아직 없어 설교 노트를 만들 수는 없다 — 이번 변경은 ②가 붙는 즉시 작성·수정이 동작하도록 한 **계약·인프라 완성**. ②는 `NoteEditArgs(category:'SERMON', verseIds:[...])` 한 줄로 연결.
- **QT(MEDITATION) 수정·4섹션 데드코드·N-04 표시 불일치**는 Lead 확인 필요한 제품 결정 → 별도 항목.
- @멘션 verseId는 "삽입 시점" 누적이라 본문에서 텍스트를 지워도 참조는 유지(§6.4.1 의도와 일치). 정밀한 "삭제 시 해제"는 후속.
