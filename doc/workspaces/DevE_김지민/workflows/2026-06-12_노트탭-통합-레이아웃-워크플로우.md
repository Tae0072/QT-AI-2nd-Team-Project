## 작업 목표 (승인 플랜 §⑤⑧)

노트 탭(N-01)을 **목록↔달력 토글 없는 단일 통합 화면**으로 재구성한다.

```
[ 상단 캘린더(월/2주) ]      ← ④⑥⑦ MeditationCalendarView 재사용, 항상 표시
─────────────────────────
[                  상태 ▾ ]  ← 상태 필터(카테고리 칩 위, 우측 드롭다운)
[ 카테고리 칩: 전체 + QT/설교/기도/회개/감사 → ]   ← 풀폭 가로 스크롤
─────────────────────────
[ 노트 목록 ]
                  ( ＋ )      ← FAB: 평소 작은 원(N-02)
              ( ＋ 기도 작성 ) ← 기도/회개/감사 칩 선택 시 알약으로 변신(N-03 직행)
```

> 레이아웃은 여러 차례 UX 검토 끝에 위 형태로 확정(2026-06-12). 상태 드롭다운은 칩과 같은 줄에 두면 폰에서 비좁고, 칩 아래 인라인 작성 줄은 어색해 **둘 다 빼고** 필터는 칩 위 우측, 작성은 FAB로 모았다.

- ⑧ "종류별 분리" = **카테고리 칩 필터**, "임시저장 따로 보기" = **상태 필터(우측 드롭다운, 칩 위)**. 별도 탭·섹션은 만들지 않는다.
- ⑤ 빠른 작성 = **맥락형 FAB**(지윤님 "칩 선택 시 작성 노출" 깔끔함 + 발견성 문제를 동시 해결): FAB를 항상 노출(발견성)하되 기도/회개/감사 칩 선택 시 `＋ {카테고리} 작성` 알약으로 변신해 **N-02 생략하고 N-03 직행**(`NoteEditArgs(category)` 고정 전달). QT·설교/전체 선택 시엔 작은 `＋` 원 → N-02. **인라인 작성 줄은 두지 않는다**(어색·레이아웃 흔들림 제거).
- FAB 스타일: Material 규격 높이 48dp 제약을 피해 **커스텀 알약(Material+InkWell)** 으로 만들고, 색은 페일 블러시(#EADAD2) 채움 + 다크 브라운(#4A3B2E) 글자로 Calm Paper 톤에 맞춤(라이트/다크 공통 고정색, 전역 FAB 토큰 미변경). 색·크기는 화면 상수(`_kFabBg`/`_kFabFg`)·패딩으로 조절.

## 현재 구조 (착수 시점, dev)

`note_list_screen.dart`:
- AppBar에 **목록↔달력 토글 IconButton** (`noteCalendarViewProvider`, 기본 `true`=달력 먼저).
- `showCalendar==true` → `MeditationCalendarView` + `_NoteListBody` (이때 **칩이 안 보임**).
- `showCalendar==false` → 카테고리 칩(전체+5종) + `_NoteListBody`.
- 카테고리 칩(`_CategoryChip`) + `noteCategoryFilterProvider`(null=전체)는 **이미 존재** → 재사용.
- FAB → `AppRouter.noteCategorySelect`(N-02).

`note_providers.dart`:
- `noteCategoryFilterProvider` (StateProvider<String?>) — 있음.
- `noteCalendarViewProvider` (StateProvider<bool>) — **제거 대상**.
- `notesProvider` (FutureProvider.autoDispose) — `noteCategoryFilterProvider`만 watch.

`note_repository.dart`:
- `getNotes({String? category, int page})` — **`status` 파라미터 없음** → 추가 필요.

`note_category_select_screen.dart` / `note_edit_screen.dart`:
- N-02는 `NoteEditArgs(category: code)`로 N-03 진입. **칩 컨텍스트 작성도 동일 계약(`NoteEditArgs(category)`) 재사용** → N-03 수정 불필요.

## 파일별 변경

### 1. `note_providers.dart`
- **삭제**: `noteCalendarViewProvider`.
- **추가**: `noteStatusFilterProvider = StateProvider<String?>((ref) => null)` (null=전체, `'DRAFT'`=임시저장, `'SAVED'`=저장).
- **수정**: `notesProvider`가 `noteStatusFilterProvider`도 watch → 상태 변경 시 자동 재조회. `getNotes(category:, status:)` 전달.

### 2. `note_repository.dart`
- `getNotes`에 `String? status` 파라미터 추가 → `if (status != null) 'status': status` 쿼리 첨부. (04 §4.3.1: `GET /notes?status=SAVED` 지원)

### 3. `note_list_screen.dart`
- AppBar 토글 `IconButton`·`showCalendar` 분기 **제거**.
- body = 단일 `Column`: `MeditationCalendarView()`(항상) → `Divider` → **상태 드롭다운(`_StatusFilterDropdown`, 우측 정렬)** → 카테고리 칩(풀폭 가로 스크롤) → `Divider` → `Expanded(_NoteListBody())`.
- `_StatusFilterDropdown`(신규): `DropdownButton<String?>`(null=전체/DRAFT/SAVED) → `noteStatusFilterProvider`.
- **맥락형 FAB**: `quickCreateCategory`(= `writableNoteCategories`에 속한 선택 카테고리)가 있으면 커스텀 알약(`Material+InkWell`, `NoteEditArgs(category)`로 N-03 직행), 없으면 `FloatingActionButton.small`(N-02). 인라인 작성 줄·`noteCalendarViewProvider`는 제거.
- FAB 색 상수 `_kFabBg`(#EADAD2)/`_kFabFg`(#4A3B2E) — 화면 전용, 전역 FAB 토큰 미변경.

### 4. l10n (`app_ko.arb` / `app_en.arb` + 생성)
- `noteStatusAll`("전체"), `noteStatusDraft`("임시저장"), `noteStatusSaved`("저장").
- `noteQuickCreate`("{label} 작성", placeholder `label`). — 기존 `noteFilterAll`은 카테고리용으로 이미 있음, 상태용은 별도 키.

## 주의사항

- **달력은 제거가 아니라 "토글 없이 항상 상단"**이다. ④⑥⑦의 `meditation_calendar.dart`(월/2주 토글·색점·`_devMockDays`)는 손대지 않고 그대로 끼운다.
- `noteCalendarViewProvider`를 참조하는 **테스트·다른 화면이 있는지 grep** 후 제거(누락 시 컴파일 깨짐).
- 칩 컨텍스트 작성은 **N-03 진입 계약(`NoteEditArgs(category)`)을 ②⑪과 공유**한다(설교 노트 작성도 같은 메커니즘으로 통일 예정). N-03(`note_edit_screen.dart`)은 수정하지 않는다.
- base는 **`origin/dev` 최신**. 로컬 `dev`(갈라진 낡은 라인)를 머지하지 말 것 — 대량 충돌. 잘못 머지 시 `git merge --abort`.
- 상태 필터 enum은 서버 `notes.status` = `DRAFT`/`SAVED`만 사용(04 §8.2). `DELETED`는 목록에서 제외되므로 필터 대상 아님.
