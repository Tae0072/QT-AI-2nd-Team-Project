## 스코프 (확정: A1)
- **대상**: study 도메인 `SimulatorClip`(대시보드 "시뮬레이터 MISSING"이 가리키는 것). qtvideo(QtVideoClip)는 **별도 브랜치 B**.
- **A1 = 조회 + 숨김** MVP. **게시(Publish, AI 산출물 sceneScript 연동)는 후속**(AD-03 연동).

## 현황
- `SimulatorClip`·study 어드민 UseCase(Get/Publish/Hide)는 service-bible(원본)+admin-server(복사) **양쪽 존재**하나 **admin 컨트롤러·화면 없음**, **목록 쿼리 없음**.
- 상태: `PENDING/APPROVED/REJECTED/HIDDEN`. hide 입력 = `aiAssetId`.

## 작업 (파일 계획)
### BE service-bible(원본) → admin-server(동기화)
1. `SimulatorClipRepository`: 목록 쿼리(optional status·qtPassageId, 페이지) 추가.
2. `ListAdminSimulatorClipsUseCase` + `ListAdminSimulatorClipsQuery` + `AdminSimulatorClipListResponse/Item`(id, qtPassageId, title, status, aiAssetId, approvedAt).
3. 구현 서비스(기존 `SimulatorClipPublishService` 또는 신규 query 서비스)에 list 구현.
### admin-server 단독
4. `AdminSimulatorClipController`: `GET /api/v1/admin/simulator-clips?status=&qtPassageId=&page=&size=`(list), `POST /api/v1/admin/simulator-clips/{aiAssetId}/hide`(hide). 권한 requireReviewer(REVIEWER/SUPER_ADMIN).
5. hide 감사로그: admin-server orchestrator가 `HidePublishedSimulatorClipUseCase` + `WriteAuditLogUseCase`(action `SIMULATOR_CLIP_HIDE`, target `AI_GENERATED_ASSET`=aiAssetId).
6. `AuditQueryService.AI_ACTION_TYPES`에 `SIMULATOR_CLIP_HIDE` 추가(target=AI_GENERATED_ASSET이라 기존 AI 감사 뷰에 노출).
### FE admin-web
7. `api/simulatorClips.ts`: listSimulatorClips, hideSimulatorClip.
8. `SimulatorClipsPage`(AD-style 목록: 본문·제목·상태·승인시각 + 상태 필터 + 숨김 액션) + 라우트 + 네비.
9. 대시보드 시뮬레이터 CTA(`#647` 임시 `/qt-passages?focusId=`)를 `/simulator-clips`로 교체.
