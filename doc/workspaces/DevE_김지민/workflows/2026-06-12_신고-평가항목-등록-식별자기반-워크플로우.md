## 배경 / 결정 (피드백 수용)

평가 항목 생성의 JSON 입력에 대해 다음 원칙을 수용:
- **FE는 판단값만 보낸다**(원문/프롬프트/민감정보 미전송).
- **백엔드가 식별자·메타로 `inputJson`을 조립**한다.
- 신고에서 만든 케이스는 `sourceType=USER_REPORT`, `sourceId=reportId`.

진단:
- asset→candidate는 이미 메타만 저장(원칙 부합).
- 수동 폼은 raw `inputJson`을 받아 위반 → 식별자 기반으로 전환.
- 신고→케이스 엔드포인트 부재 → 신규 백엔드(asset→candidate mirror).

**수동 추가 방식 결정**: 제거(A)가 아니라 **식별자 기반 수동 추가(B)** 채택 — 손으로도 추가하되 자유 텍스트 없이
대상 식별자(유형·ID)+기대판정만. (Lead/사용자 합의 2026-06-12)
