## 1. 문제
PR #649가 추가한 관리자 액션 `AI_EXPLANATION_GENERATE_REQUEST`(target=`QT_PASSAGE`)는 감사 로그에 기록되지만, AD-07 감사 조회(`AuditQueryService`)가 `targetType = AI_GENERATED_ASSET` **단일 등치 필터**로 고정돼 있어 추적 화면에 노출되지 않았다(리뷰 WARN). F-06 "관리자 작업 추적" 미완.

## 2. 해결 (백엔드 전용, FE 변경 없음)
1. `AuditQueryRepository.LIST_WHERE`: `and auditLog.targetType = :targetType` → **`and (:targetType is null or auditLog.targetType = :targetType)`** (targetType null = 대상 무관). 하위호환: non-null이면 기존과 동일.
2. `AuditQueryService`:
   - `AI_ACTION_TYPES`에 `AI_EXPLANATION_GENERATE_REQUEST` 추가.
   - `resolveTargetType`: 미지정 시 `AI_GENERATED_ASSET` 강제 → **null(대상 무관 조회)**. 지정 시 허용집합 `{AI_GENERATED_ASSET, QT_PASSAGE}` 검증.
3. FE(AD-07 `AuditLogsPage`): **변경 불필요** — targetType을 보내지 않고(기본 null) actionType은 자유 입력이라, 기본 목록이 자동으로 QT_PASSAGE 해설 생성 로그까지 노출(컬럼 "QT_PASSAGE #35").

함께(같은 #649 후속): 시드 서비스의 `context==null` 죽은/모순 분기 제거(양쪽) — getContentContext는 미존재 시 예외를 던지므로 soft-result 분기 불필요.
