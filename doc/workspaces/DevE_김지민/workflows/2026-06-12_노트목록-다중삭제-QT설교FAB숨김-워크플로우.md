## 배경 / 결정

- **①** 노트 삭제 API(`DELETE /notes/{id}`)는 있으나 N-04 단건 삭제만 쓰고 **목록(N-01)에 선택·삭제 UI가 없었다.**
- **②** 기록 탭의 작성 FAB이 QT/설교 칩에서도 `+`(→N-02)로 떠 어색했다. QT·설교는 각자 탭(오늘의 QT / 성경)에서
  작성하는 게 맞으므로, **기록에선 QT/설교 = 수정·삭제만** 으로 정리(작성=각 탭, 수정·삭제=기록).

## 변경 내용

### ① 다중 선택 삭제 (AppBar ☰ 토글)
- `note_providers`: `noteSelectionModeProvider`(bool) + `noteSelectedIdsProvider`(Set<int>) 추가.
- `note_repository`: `deleteMany(ids)` — bulk API가 없어 단건 DELETE 반복, **실패 id 목록 반환**(부분 실패 안내).
- `NoteCard`: 선택 props(`selectionMode`/`selected`/`onToggleSelect`) 추가 — 선택 모드면 좌측 체크 아이콘 +
  선택 강조 테두리, 탭=선택 토글.
- `note_list_screen`:
  - AppBar에 **☰ 선택** 액션 → 선택 모드 진입. 선택 모드 AppBar = **✕(종료) + "n개 선택" + 전체선택(토글)**.
  - 하단 **삭제 바**(`_SelectionDeleteBar`, 선택 0개면 비활성) → 확인 다이얼로그 → `deleteMany` → 목록·달력
    invalidate → 모드 종료 → 결과 스낵바(부분 실패 시 안내).

### ② QT/설교 FAB 숨김 + 빈 상태 안내
- FAB을 `showFab = !selectionMode && category != MEDITATION && category != SERMON`로 가드 →
  **QT/설교 칩·선택 모드에서는 작성 FAB 숨김**. 전체·기도·회개·감사는 유지.
- 빈 목록일 때 카테고리별 안내: QT → "오늘의 QT 화면에서 작성", 설교 → "성경 화면에서 작성"(작성 유도).

- 빈 상태 안내는 **텍스트 유도**까지(탭 이동 버튼은 미구현 — 탭 전환 결합을 피함). 필요 시 후속.
