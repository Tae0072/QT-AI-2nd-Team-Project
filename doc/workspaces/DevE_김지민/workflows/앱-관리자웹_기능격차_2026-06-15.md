# 앱 ↔ 관리자웹 기능 격차 정리

기준: dev 브랜치 + 관리자웹 작업트리(WIP 포함) · 작성 2026-06-15

관리자웹에는 있으나 앱에 미구현인 것, 앱에는 있으나 관리자웹에 관리 화면이 없는 것을 정리. AD-01~11은 dev 반영, AD-12·13·15·16·17·18·19는 다른 세션 WIP(미머지).

> **진행 상황(2026-06-15 갱신):** 대시보드(AD-01) 문제 상태 조치 동선은 **PR #647**로 완료. **관리자 해설 생성 트리거**는 `seedForPassage` 기반 admin API + 감사로그 + 대시보드 버튼까지 반영 확인 완료(#649). **시뮬레이터(QT 영상) 전용 관리 화면(B-1, 조회+숨김)**은 **PR #652**(AD-14)로 완료되어 대시보드 시뮬레이터 CTA도 전용 화면으로 교체됨. ⚠️ 다만 **시뮬레이터 승인→게시 자동 배선은 아직 미구현**이다 — 게시/숨김 서비스(`SimulatorClipPublishService`)는 존재하나 `AiAssetReviewService` 승인 흐름은 EXPLANATION+BIBLE_VERSE만 게시하고 SIMULATOR 분기가 없어, 승인본이 자동으로 `simulator_clips` 공개 모델에 반영되지는 않는다(후속 B).

## A. 관리자웹엔 있는데 앱엔 미구현


| 기능(관리자 화면)                          | 관리자웹           | 앱              | 설명 · 필요 조치                                                      |
| ----------------------------------- | -------------- | -------------- | --------------------------------------------------------------- |
| 미션 (AD-16, WIP)                     | 있음(미션 정의·관리)   | 없음             | 앱에 사용자 미션(목록·수행·달성) 화면이 없음. 관리자만 정의 가능. → 앱 미션 화면 신설 필요(추진 시).  |
| 앱 업데이트 안내 (AD-19 + appversion, WIP) | 있음(버전·업데이트 관리) | 없음(개발자 버전 타일만) | 앱에 "업데이트 필요/권장" 안내·버전 체크 화면 없음. → 앱 시작 시 최소버전 체크/업데이트 안내 화면 필요. |


> ※ 대시보드·감사로그·AI 모니터링/체크리스트/배치로그/AI 평가세트·성경 데이터 조회·자가진단(AD-01·07·08·09·10·11·17·18)은 관리자 전용 운영 도구라 앱 대응이 불필요 — 격차 아님.

## B. 앱엔 있는데 관리자웹엔 관리 화면 없음


| 기능(앱)             | 관리자웹                         | 앱         | 설명 · 필요 조치                                                                                                                                                                                 |
| ----------------- | ---------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| QT 영상(시뮬레이터)      | 있음(조회+숨김, AD-14)            | 있음(영상 재생) | **전용 관리 화면(조회·상태필터·숨김)은 PR #652로 완료**(`AdminSimulatorClipController`/`SimulatorClipsPage`), 대시보드 시뮬레이터 CTA도 전용 화면(`/simulator-clips`)으로 교체됨. ⚠️ **남은 격차**: ① 승인→게시 자동 배선(현재 `AiAssetReviewService`가 EXPLANATION만 게시, SIMULATOR 미연결 — 게시 서비스는 존재) ② 게시(Publish) 운영 화면/동선. SIMULATOR 생성은 비활성이라 산출물은 시스템 등록 경로로만 유입. → 후속 B. |
| AI 해설 생성(미생성 본문)  | 있음(대시보드 생성 요청 + admin API) | 해설 조회     | **격차 해소 확인(2026-06-15).** 특정 QT 본문 대상 `seedForPassage` 기반 admin API(`POST /api/v1/admin/ai/qt-passages/{qtPassageId}/explanations/generate`)와 감사로그(`AI_EXPLANATION_GENERATE_REQUEST`), 대시보드 "해설 생성 요청" 버튼 연결 확인. |
| 기록(QT노트·설교노트·손그림) | 없음                           | 있음(핵심 기능) | 사용자 비공개 콘텐츠라 설계상 제외 가능. 단 나눔으로 공개된 글은 AD-13에서 관리. → 정책 확인.                                                                                                                                 |
| TTS 본문 음성         | 없음                           | 있음(생성·캐시) | 자동 캐시라 관리 불필요할 수 있으나, 재생성·삭제 운영 필요 시 화면 검토.                                                                                                                                                |
| 홈 "오늘의 말씀" 배경     | 없음                           | 있음(랜덤 노출) | 자작 정적 자원이면 불필요. 교체·로테이션 운영 필요 시 검토.                                                                                                                                                        |


> ※ 댓글 관리: 앱은 댓글 작성·신고 가능하며, 관리자 "신고 처리(AD-04)"가 댓글(COMMENT) 대상 신고를 함께 처리 — 별도 격차 아님.

## 우선 검토 제안

① **관리자 해설 생성 트리거** — ✅ 반영 확인 완료. 대시보드 해설 CTA가 "검증" 보조 동선과 함께 "생성 요청"을 제공.
② **QT 영상(시뮬레이터) 관리자 화면(B-1)** — ✅ 조회+숨김 화면(AD-14)·대시보드 CTA 교체 완료(#652). 남은 후속(B): 승인→게시 자동 배선(`AiAssetReviewService` SIMULATOR 분기) + 게시(Publish) 운영 동선.
③ **앱 업데이트 안내(A-2)** — 배포 운영 필수.
④ **미션(A-1)** — 제품 로드맵 결정 후 진행.
