## 1. 기존 백엔드 패턴 (이걸 복제)

- **3겹 레이어**: `Controller(web) → UseCase(api 인터페이스) → Service(internal, @Transactional) → Repository`. Controller가 Repository 직접 호출 ❌.
- **권한 2단**: JWT `ROLE_ADMIN`(1단) + `admin_users.admin_role`(2단). `AdminAiAuthentication.requireXxx()` 식 헬퍼 패턴.
- **응답 봉투**: `lib-common` `ApiResponse.success(data)` 로 감쌈.
- **감사 로그**: `WriteAuditLogUseCase.write(req)` 를 같은 `@Transactional` 안에서 호출(actor·actionType·targetType·before/after JSON).
- **마이그레이션**: `db/migration/V{N}__*.sql` (Flyway, 최신 **V29**). 새 건 `V30`, `V31`.
- **본보기 코드**: 집계=`AdminAiMonitoringController`, CRUD+발행+감사로그=`domain/report`.

## 1-1. MSA 안전성

### 🔴 AD-06 공지 (`domain/notification` 하위 — B안)
- **신규**: `V30__create_notices.sql` · `notification/internal/Notice.java`(Entity) · `notification/internal/NoticeStatus.java`(enum DRAFT/PUBLISHED/HIDDEN) · `notification/internal/NoticeRepository.java` · `notification/internal/NoticeService.java`(write,@Transactional) · `notification/api/ManageNoticeUseCase.java`·`ListNoticesUseCase.java`(+DTO) · `notification/web/AdminNoticeController.java`

- **의존**: 전체 회원 id 목록(member 협의) · `WriteAuditLogUseCase`
- **API**: `GET/POST /admin/notices` · `PATCH /{id}` · `POST /{id}/publish|hide` · 권한 OPERATOR/SUPER_ADMIN
- **상태코드**: 생성 201 / 수정·발행 200 / 숨김 204

### 🟡 F1 카카오 웹 로그인 정식화 (admin-web 프론트 + 백엔드 설정)
> ✅ **2026-06-10 확정**: MSA 분리로 카카오 인증이 service-user(이승욱) **신규 `admin/auth/kakao`로 확정**(06-09 ③A 대체). 응답 형태 5개 합의 완료([계약 문서](2026-06-10_admin-kakao-auth-contract.md)).
- **목표**: 관리자 웹 로그인을 dev 토큰 수동입력 → **진짜 카카오 로그인**으로 정식화.
- **엔드포인트(✅ 확정)**: 신규 `POST /api/v1/admin/auth/kakao`(2026-06-10). 서버 `/oauth2/**` 미사용(§5 규칙 유지).
- **프론트(admin-web)**: 카카오 JS SDK 로그인 → 카카오 access token → (위 경로) 호출 → 서비스 JWT 저장 → `LoginPage`/`auth/*`의 dev 토큰입력 교체. 비관리자 로그인 시 안내(계약 §4 `ADMIN_USER_NOT_FOUND` 403, ErrorCode 그대로 표시).
- **백엔드**: 경로 확정 후 service-user에서 구현(이승욱 task 3). admin 웹 출처 CORS 허용 확인.
- **정리 대상**: DRAFT PR #319(`web-kakao-login-draft`)는 **B안(/oauth2)** 이라 교체/폐기.
- **성격**: admin-web FE 작업 중심. 브랜치 `feature/admin-web-kakao-login`.
