**확인 절차:**
1. `npm run dev` → 브라우저 `localhost:5173`
2. 로그인 화면에 ADMIN 토큰 붙여넣기(**`Bearer ` 빼고 `eyJ`로 시작하는 값만**)
3. 좌측 사이드바 **감사 로그** → `/audit-logs`
4. 확인 포인트:
   - [ ] 위에 필터(주체·행위·기간) + [조회] 버튼 보임
   - [ ] 아래 표(시각·주체·행위·대상) 보임 (데이터 없으면 빈 표 — 정상)
   - [ ] [조회] 누르면 필터 반영 / 페이지 이동 동작
   - [ ] 실패 시 우측 빨간 토스트로 메시지

**⚠️ "인증이 필요" / 로그인으로 튕김 (토큰 문제, 코드 아님):**
- ① **`Bearer ` 같이 붙여넣었는지** — 코드가 자동으로 붙이므로 `eyJ...`만. (F12 → Application → Local Storage → `qtai_admin_token` 확인)
- ② **토큰 만료**(보통 30분) → 새로 발급
- ③ F12 → Network → `audit-logs` 요청 Status: **401**=토큰 문제(①②) / **403**=그 회원이 관리자 권한(OPERATOR/REVIEWER/SUPER_ADMIN) 없음
- ※ 카카오 웹 로그인(정식 토큰 발급 경로)은 강태오 담당 — 그 전엔 수동 토큰 필요
