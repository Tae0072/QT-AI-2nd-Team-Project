## 3. 남은 리스크 / 후속

- **시뮬레이터 전용 관리 화면 부재(기능격차 B-1)**: 현재 시뮬레이터 CTA는 임시로 `/qt-passages?focusId=`로 연결. 후속 PR에서 전용 화면 신설 후 CTA 교체 예정.
- **해설 수동 생성 동선 부재**: AD-03(`/ai-assets`)은 승인/반려/숨김/재생성(검증)만 가능하고, 미생성 해설을 처음부터 만드는 "생성 요청" 버튼이 없음(생성은 배치/시스템 작업). 그래서 이 PR의 해설 CTA 라벨을 "해설 검증하러 가기"로 정직화. **후속 PR `feature/admin-ai-explanation-generate`(가칭)에서 관리자 해설 생성 트리거(BE 엔드포인트+감사로그+FE 버튼) 추가** 예정 — 시뮬레이터 관리 화면과 함께 "관리자 생성 동선" 묶음.
- **해설 CTA**는 화면(`/ai-assets`)까지만 정밀, verse 단위 targetId 필터는 후속.
- AD-08은 모니터링 실데이터가 비어 있어 표시 **구조** 개선 위주로 검증(실데이터 회귀는 AD-04/06/07/10 QA 별도).

## 4. 사용자 확인 체크리스트 (로컬 admin-web)

> ⚠️ admin-web 자체 로그인이 현재 403(별도 이슈)이라 로그인으로는 못 들어갈 수 있음. 그 경우 같은 RS256 키로 ADMIN 토큰을 발급해 `localStorage["qtai_admin_token"]`에 주입하면 진입 가능.

|---|---|---|---|
| 1 | 대시보드 | 운영 카운트 3칸(AI 검증 대기·신고 접수·신고 검토 중) **높이 동일** | ✅ 같은 높이 |
| 2 | 대시보드 | AI 검증 대기 값>0 → "검증하러 가기 →" 노출, 클릭 | `/ai-assets` 이동(VALIDATING 목록) |
| 3 | 대시보드 | 신고 접수·검토 값이 0이면 링크 **없음** / 값 있으면 "신고 처리하기·검토 이어가기" | 클릭 시 `/reports?status=` 필터 진입 |
| 4 | 대시보드 | 오늘 QT 상태 ≠ READY → "QT 관리" 링크 | `/qt-passages?focusId=` |
