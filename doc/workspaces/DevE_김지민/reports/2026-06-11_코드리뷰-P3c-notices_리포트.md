## 변경 내용 (렌더 B — 풀 CRUD)

AD-06 공지의 임시 타입 `[key:string]:unknown` + generic 렌더를 백엔드 계약 기준 실타입 + 풀 CRUD로 전환.

- `src/api/notices.ts`:
  - 실타입 `Notice`(목록, `bodyPreview`)·`NoticeDetail`(상세, `body`)·`NoticeStatus` 3종(DRAFT/PUBLISHED/HIDDEN)·`NoticeRequest`·`NoticePublishResponse`(`notificationResult`).
  - `createNotice`(POST 201)·`updateNotice`(PATCH 200)·`publishNotice`(POST 200, 발행 결과)·`hideNotice`(POST **204**, envelope 없음 → unwrap 미사용) + 기존 list.
- `src/pages/NoticesPage.tsx`:
  - `usePagedList` 목록 + 명시 컬럼(제목·미리보기·상태 Tag(한글)·발행시각) + 페이징. (공지 목록은 필터 없음 — 정렬 `createdAt,desc` 고정)
  - 행 액션: DRAFT→수정·발행 / PUBLISHED→숨김 / HIDDEN→없음 (Popconfirm).
  - **발행 성공 시 알림 결과** 메시지(`알림 N건 생성[, M건 실패]`).
  - 등록/수정 Modal 폼(제목 Input + 본문 TextArea), **plain text 검증**(`<`,`>` 거부 — 서버 정책과 일치).
  - "준비 중" Alert 철거 → 에러 Alert + 재시도. antd 함정 보정(Modal `forceRender`, validateFields 분리) P3b 패턴 재사용.

## ⚠️ 알려진 한계 / 후속 (중요)

- **수정 시 본문 전체를 불러올 수 없음**: 백엔드에 공지 **상세 GET이 없고**(`GET /admin/notices/{id}` 미제공, 강상민 리포트 §비고), 목록은 `bodyPreview`(잘린 미리보기)만 제공. 잘린 미리보기를 폼에 채우면 저장 시 **본문이 잘려 데이터 손실**이 나므로, 수정 모달은 본문을 빈 칸으로 두고 "다시 입력" 안내 + 기존 미리보기를 경고 박스로 표시한다(안전 우선).
  - **후속 제안**: 강상민님께 `GET /api/v1/admin/notices/{id}` 상세 엔드포인트 요청 → 받으면 수정 시 본문 prefill로 UX 개선(별도 PR).
