## 한 일

### #1 노트 진입 시 달력이 기본 + 하단 목록
- `noteCalendarViewProvider` 기본값 `false`(목록)→`true`(달력) → 노트 탭 진입 시 달력이 첫 화면.
- 달력 모드 = **달력(위) + 노트 목록(아래)** 한 화면. 목록을 `_NoteListBody` 공용 위젯으로 추출해 달력/목록 모드가 공유.
- 레이아웃: 달력 고정 높이 + 목록 `Expanded` 스크롤. 달력은 좌우 스와이프(월 이동), 목록은 상하 스크롤이라 제스처 충돌 없음.
- `MeditationCalendarView`를 **항상 렌더(고정 높이)** 구조로 리팩터(점 데이터만 `maybeWhen`으로 채움) → Column에 목록을 붙일 수 있게.

### #2 날짜 탭 → 5종 체크리스트 (표시 전용)
- 신규 [note_day_checklist_sheet.dart](../../../../flutter-app/lib/features/note/widgets/note_day_checklist_sheet.dart): 날짜 탭 → 바텀시트에 설교·묵상·기도·회개·감사 표시, `CalendarDay.categories`에 있으면 ✓.
- `_onDayTap`의 기존 "묵상 노트 상세 이동"을 체크리스트 시트로 교체.
- **자동 ✓ 배선**: 작성([note_edit_screen](../../../../flutter-app/lib/features/note/screens/note_edit_screen.dart) `_save`)·삭제([note_detail_screen](../../../../flutter-app/lib/features/note/screens/note_detail_screen.dart))에 `ref.invalidate(meditationCalendarProvider)` 추가 → 평소 작성하면 달력 갱신되며 자동 체크.
- **수동 체크는 미채택**: ✓는 "실제 작성함"을 뜻하고, 수동 체크 상태를 저장할 DB 필드가 없어 새로고침 시 사라짐 → 작성=자동 ✓(표시 전용)로 일관 유지.
