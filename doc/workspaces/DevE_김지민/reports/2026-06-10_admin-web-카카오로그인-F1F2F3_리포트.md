## 1. 개요 (무엇을 왜)

MSA가 service-user/admin-server로 더 분리되면서 내 역할이 **관리자 백엔드 구현자 → admin-web(FE) 구현자**로 바뀌었다(배경은 워크플로우 §0). 그 첫 작업으로 **관리자 로그인을 카카오 JS SDK 방식으로 정식화**하는 F1·F2·F3을 구현했다.

- **결정(2026-06-10)**: ① 카카오 = JS SDK(서버 `/oauth2` 미사용) · ② 관리자 인증 = 카카오 단일 · ③ admin-web = 처음부터 8090(admin-server) 기준.
- 카카오 인증 응답 형태 5개는 이승욱과 **합의 완료**, 그 계약대로 FE를 구현했다.

## 2. 한 일 (커밋 범위)

| 작업 | 파일 | 내용 |
|---|---|---|
| **F3** env 8090 | `admin-web/.env.example`, `vite.config.ts`, `src/config/env.ts` | 프록시 기본값 `8080→8090`, `VITE_KAKAO_JS_KEY` 추가/`KAKAO_JS_KEY` export |
| **F1** 카카오 버튼 | `index.html`, `src/auth/kakao.ts`(신규), `src/pages/LoginPage.tsx` | 카카오 JS SDK CDN 로드 + `loginWithKakao()` 헬퍼 + "카카오로 로그인" 버튼 |
| **F2** 토큰 교환 | `src/api/adminAuth.ts`(신규), `LoginPage.tsx` | `POST /api/v1/admin/auth/kakao` → ADMIN 토큰 저장 |

## 3. 구현 흐름

```
[카카오 로그인 버튼 클릭]
  → loginWithKakao()           # src/auth/kakao.ts: window.Kakao.Auth.login → 카카오 access token
  → loginAdminWithKakao(token) # src/api/adminAuth.ts: POST /admin/auth/kakao { kakaoAccessToken }
  → login(res.accessToken)     # AuthContext: ADMIN accessToken 저장 = 로그인
  → /admin/me 로 adminRole 확인 # 기존 AuthContext 로직 재사용
  → 대시보드로 이동
```

## 4. 합의 5개를 코드 어디에 반영했나

| # | 합의 | 코드 반영 |
|---|---|---|
| ① | 응답 키 `admin` 블록 | `AdminLoginResponse.admin` 타입으로 파싱 |
| ② | `adminRole` 단일 문자열 | `adminRole: AdminRole`(유니온 문자열) |
| ③ | `refreshToken` body | `AdminLoginResponse.refreshToken: string`(쿠키 미사용) |
| ④ | 403 ErrorCode 그대로 표시 | `catch`에서 `[${e.code}] ${e.message}` Alert 표시 |
| ⑤ | access 30분/refresh 14일 | 계약 주석 명시, 만료 시 401→재로그인(기존 client 인터셉터) |

## 5. 공부한 것 (개념 정리)

- **카카오 로그인이 서버를 거치지 않는 이유**: JS SDK가 브라우저에서 카카오 access token을 직접 받고(`Kakao.Auth.login`), 우리 서버에는 그 토큰만 넘긴다. 서버사이드 리다이렉트(`/oauth2`)가 필요 없어 규칙(§5)과도 안 맞부딪힌다. 앱(Flutter)도 같은 방식이라 일관적.
- **"계약 소비자(FE)"가 된다는 것**: 백엔드가 없어도 **합의된 응답 형태(계약)** 만 있으면 FE 타입·호출 코드를 미리 다 짤 수 있다. 실제 호출만 백엔드가 뜬 뒤 된다. → 오늘처럼 백엔드 전이라도 typecheck·build까지 완성 가능.
- **Vite 환경변수/프록시**: `VITE_` 접두사 변수만 프런트에서 읽힌다(`import.meta.env`). 개발 중 CORS를 피하려고 `/api`를 `vite.config.ts` proxy로 백엔드(8090)에 전달한다. `.env`는 git에 안 올리고 `.env.example`만 템플릿으로 둔다.
- **외부 SDK의 TS 타입**: 전체 타입 패키지를 깔지 않고, 쓰는 부분만 `declare global { interface Window { Kakao?: ... } }`로 **최소 선언**했다(`src/auth/kakao.ts`). strict 모드에서도 안전.
- **dev 전용 분기**: `import.meta.env.DEV`로 감싸면 개발 빌드에만 들어가고 prod 빌드에선 사라진다. → "임시 토큰 입력"을 prod에선 제거(요구 충족)하면서 로컬 작업은 안 막는 절충.
