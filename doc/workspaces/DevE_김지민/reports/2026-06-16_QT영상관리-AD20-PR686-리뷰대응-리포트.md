## 1. 한 줄 요약

앱 재생용 QT 영상 관리(AD-20)를 완성하고, 3회차에 걸친 리뷰 지적을 모두 반영했다. 관리자가 등록한 실제 mp4가 유저 API `/api/v1/qt/{id}/video`에서 `READY`로 반환되는 것까지 로컬 검증했고, CI 게이트를 로컬에서 그대로 통과 재현했다.

## 2. 기능 범위

- **원본 영상**: 성경권 단위 영상 URL·길이 등록/수정/조회, 활성 1개 정책.
- **절별 구간**: `bible_verse_video_segments`에 절↔타임코드 매핑 저장(전체 교체).
- **QT 클립**: 본문 절 범위에서 `min~max` 자동 계산해 `qt_video_clips` 생성(`prepareClip`), 상태 변경(APPROVED/HIDDEN/FAILED), 삭제.
- **직접 클립 생성(하이브리드)**: 절별 구간 없이 시작/끝 초만으로 클립 생성·교체(`POST /clips/manual`).
- **유저 노출**: service-bible `GET /api/v1/qt/{id}/video`가 활성 APPROVED·미삭제 클립을 `READY`로 반환 → Flutter `video_player` 재생.

## 3. 디버깅 (라이브 DB/API 원인 규명)

- **원본 수정 400(`C0002`)**: 프런트가 수정 시 disabled `bibleBookId`를 전송 → 서버 unknown 속성 거부. update payload에서 제거해 해결.
- **구간 2건만 저장**: 버그 아님 — 기본 예시 템플릿을 그대로 저장한 것. 템플릿 그대로 저장 방지 가드 추가.
- **클립 생성 404/“생성 못 함”**: 본문 ID 오입력(404) 또는 절 구간 미완성(200 prepared:false). id=2 정입력 시 `prepared:true` 확인.
- **클립 목록 `#2` 혼란**: `클립 ID / QT 본문(날짜) / 원본 영상 ID`로 라벨 분리.

## 5. 핵심 설계 결정

- **삭제 = soft-delete + cascade**: 프로젝트 공통 정책(노트·나눔)과 통일. 행 보존·복구 가능, 목록은 `deleted_at IS NULL` 필터. 활성 선택은 `active_unique_key`로 자동 제외.
- **감사 로그 원자성**: 컨트롤러 후처리 → 서비스 트랜잭션 내부 기록으로 이동(`MusicTrackService` 패턴). 변경 실패 시 감사도 롤백.
- **하이브리드 클립 생성**: 절별 구간(책 전체 1회 입력→재사용) + 직접 시작/끝(단건 빠른 지정) 병행.
- **절↔초 자동 매핑은 후속**: 글자수 가중 분할 등. `2026-06-16_qtvideo-auto-verse-timecode-후속.md`에 수정 지점·한계 기록.
