## 2. 구현한 3기능 (전부 USER 작성자 본인만)

| 동작 | 엔드포인트 | 상태 전이 |
|---|---|---|
| 삭제 | `DELETE /api/v1/sharing-posts/{id}` | PUBLISHED·HIDDEN → DELETED (soft) |
| 숨김 | `PATCH /api/v1/sharing-posts/{id}/hide` | PUBLISHED → HIDDEN (hiddenAt=now) |
| 되돌리기 | `PATCH /api/v1/sharing-posts/{id}/show` | HIDDEN → PUBLISHED (hiddenAt=null) |

- **DELETED는 종료 상태** — hide/show 시 `409 INVALID_STATUS_TRANSITION`(엔티티가 검증).
- **멱등**: 이미 목표 상태면 서비스가 조용히 통과. **책임 분리** — 멱등=서비스, 전이검증=엔티티.
- 본인 아님 403 FORBIDDEN, 없는 글 404 SHARING_POST_NOT_FOUND.

## 4. 오늘 배운 개념

| 개념 | 한 줄 |
|---|---|
| 멱등 vs 전이검증 책임 분리 | 멱등(이미 그 상태면 통과)=서비스, 불가능한 전이(삭제→숨김)=엔티티 예외. NoteService/CommentService 패턴과 일치 |
| 엔티티 캡슐화 | status를 서비스가 직접 set하지 않고 엔티티 delete()/hide()/show()로 — "규칙을 엔티티가 지킨다" |
| 공통 헬퍼로 중복 제거 | 3메서드의 조회+본인검증을 `findOwnedPost`로 묶음 |
| findById vs findByIdAndStatus | 삭제/숨김 대상은 상태 무관 조회(findById), 피드/좋아요는 PUBLISHED 한정 |
| null vs "" / NOT NULL | (전일 학습) null은 NOT NULL 위반·크래시, ""는 통과 |
| 이미 머지된 브랜치 점검 | 어제 작업이 #191로 dev에 머지됨 확인 → 새 브랜치에서 출발(stale 21커밋 회피) |
| PATCH 의미 | 리소스 일부 상태(공개여부)만 부분 변경 → PATCH |

## 5. 환경 메모 + 스모크 결과

- 로컬 `gradlew test` 실행 불가(한글 경로+Gradle9 argfile) → **테스트는 CI(Linux)**. 로컬은 `compileTestJava`로 컴파일만 확인(`BUILD SUCCESSFUL`).
- **Docker(MySQL) 스모크 통과** — `docker compose up --build`로 전체 스택 기동(dev 프로파일=dev-bypass 자동), MySQL 시드 후 dev-console "나눔 글 관리"에서:
  - 숨김 204 → 멱등 204 → 되돌리기 204 → 타인 **403** → 없는 글 **404** → 삭제 204 → 삭제된 글 숨김 **409** ✅
- 스모크 중 배운 함정(다음 참고): Docker Desktop 먼저 켜기 / 포트 8080·3306 충돌(로컬 MySQL `Stop-Service MySQL80`) / `.env` JWT 키 빈값 시 `Missing key encoding` 부팅 실패.
- 참고: dev 프로파일이 그 사이 H2→MySQL로 분리됨(local=H2). `DevSecurityConfig`가 `@Profile("dev")`라 H2(local)에선 dev-console 인증 우회가 꺼져 있어, dev-console 스모크는 **dev 프로파일(Docker MySQL)** 로 해야 함.
