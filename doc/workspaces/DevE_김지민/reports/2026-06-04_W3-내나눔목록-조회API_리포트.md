## 1. 배경 — 왜 이 작업을 했나

V1 백엔드 점검 중, 김지민 담당 나눔 도메인에서 **유일하게 누락된 엔드포인트**를 발견.

- 이미 만든 hide/show/delete(공개중단·되돌리기·삭제, #208)의 **짝**. 목록 조회가 없으면 M-05 화면이 내 글을 띄울 수 없어 관리 버튼이 동작할 대상이 없다.
- 공개 피드 `GET /sharing-posts`(`ListSharingPostsUseCase`)는 PUBLISHED 전용이라 본인의 HIDDEN 글을 못 본다 → 별도 조회 경로 필요.
- 도메인 경계상 `sharing_posts` 조회는 나눔 도메인만 가능 → 마이페이지에 그리더라도 백엔드는 김지민이 제공해야 함(구현_리스트·워크플로우에 누락돼 있던 항목, 화면 ID 혼선 07 §19.3이 원인).

## 2. 설계 결정 (소크라테스 모드 — 선택 후 구현)

| 결정 | 선택 | 이유 |
|---|---|---|
| 응답 DTO | **신규 lean DTO** (재사용 X) | 04 §4.4.5는 공개 피드보다 4필드(nickname·verse·preview·likedByMe) 적음. 계약 정확성 + 피드와 독립 진화(결합 끊기) |
| status 기본 동작 | **생략 시 PUBLISHED+HIDDEN** (DELETED 제외) | M-05에서 숨긴 글 되돌리기(show) 하려면 목록에 보여야 함. `DELETED`·정의 외 값 → 400 |
| 검증 위치 | **Service** (`resolveStatuses`) | 레포는 받은 상태로 조회만, "DELETED 거부"는 Service 책임 → 레포 단순 유지 |
