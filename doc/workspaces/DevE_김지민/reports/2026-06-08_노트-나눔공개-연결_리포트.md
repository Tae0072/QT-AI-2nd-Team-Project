## ⚠️ 발견한 백엔드 갭 (후속 과제 — 이번 작업 범위 밖, 미수정)

**나눔 공개 후 "공유됨" 뱃지가 안 뜨고, "나눔 공개" 버튼이 안 사라진다.**

- **원인**: 서버 [SharingPostService.publish()](../../../../qtai-server/src/main/java/com/qtai/domain/sharing/internal/SharingPostService.java#L175)가 공유본(SharingPost)만 저장하고 **원본 노트의 `visibility`를 `SHARED`로 바꾸지 않음.**
- 서버는 `shared = (note.visibility == SHARED)`로 계산([NoteService.java:320](../../../../qtai-server/src/main/java/com/qtai/domain/note/internal/NoteService.java#L320)) → visibility가 안 바뀌니 `shared`는 영원히 false.
- 파급:
  1. `GET /notes/{id}`의 `shared`=false → Flutter "공유됨" 뱃지 안 뜸.
  2. `if (!detail.shared)`가 항상 true → "나눔 공개" 버튼이 공개 후에도 노출 → 재탭 시 서버 409(중복)로 막히나 Flutter엔 일반 실패로 표시.
- **핵심 기능(피드 노출)은 정상.** 위는 뱃지/버튼 숨김 등 **부가 UX만** 영향.
- **수정안(미적용)**: note 도메인에 `MarkNoteSharedUseCase`(visibility→SHARED, 멱등) 추가 + `publish()`에서 호출. note 도메인(이지윤)·sharing 도메인(김지민) 둘 다 걸쳐 **조율 필요** → 별도 백엔드 PR로 분리.
- **결정(2026-06-08)**: 이번 Flutter 브랜치엔 백엔드 안 섞음. 후속 과제로 이관.
