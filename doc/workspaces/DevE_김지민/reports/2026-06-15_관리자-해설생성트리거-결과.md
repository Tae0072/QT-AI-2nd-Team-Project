## 1. 한 일
AD-01 대시보드의 해설 "없음" 상태에서 관리자가 **그 본문의 해설 생성을 직접 트리거**할 수 있게 했다. 기존엔 미생성 해설을 만드는 동선이 없었고(생성=배치/시스템), CTA는 검증 화면 이동뿐이었다.

### 백엔드 (service-ai 원본 → admin-server 동기화)
| 변경 | service-ai | admin-server |
|---|---|---|
| `AiDailyQtVerseExplanationSeedService`: `seedForContext` 추출 + **`seedForPassage(qtPassageId, requestedBy)`** 추가(본문 단위 시딩, 기존 dedup/skip 재사용) | ✅ | ✅ 동기화 |
| `GenerateQtPassageExplanationUseCase` + Command/Result DTO | ✅ | ✅ 동기화 |
| `AiService`: UseCase 구현 — seedForPassage 위임 + 감사로그 `AI_EXPLANATION_GENERATE_REQUEST`(actor=ADMIN:{adminId}, target=QT_PASSAGE) | ✅ | ✅ 동기화 |
| `AdminAiExplanationController`: `POST /api/v1/admin/ai/qt-passages/{qtPassageId}/explanations/generate` (REVIEWER/SUPER_ADMIN, 202 Accepted) | — | ✅ admin 단독 |

> **순환 의존성 해소**: `AiService → 시드서비스 → CreateAiGenerationJobUseCase(=AiService)` 순환을 시드서비스 주입에 `@Lazy`로 끊음(양쪽 AiService).

### 프런트 (admin-web)
- `api/aiAssets.ts`: `generateQtPassageExplanation(qtPassageId)` + `GenerateExplanationResult` 타입.
- `DashboardPage.tsx`: 해설 "없음" 시 **"해설 생성 요청"** 버튼(Popconfirm → POST). 성공 시 "생성 요청됨 N건(잠시 후 검증 대기 표시)", 0건/사유 시 안내. 기존 "검증하러 가기"는 유지(이미 생성·미승인 케이스).

## 3. 남은 리스크 / 후속
- **감사 로그 조회 뷰 한계**: AD-07 `AuditQueryService`는 targetType이 `AI_GENERATED_ASSET`로 고정된 **AI 자산 전용 뷰**다. 본 액션은 target=`QT_PASSAGE`라 **기록은 되지만 이 뷰에는 안 보인다**. 감사 조회 뷰를 본문/일반 대상까지 넓히는 건 별도 변경(후속).
- **시뮬레이터(QT 영상) 전용 관리 화면(B-1)**: 팀원 분배 / 다음 브랜치.
- 대시보드 해설 CTA는 생성 요청(신규) + 검증하러 가기(기존) 2동선 유지.

## 3-1. PR 리뷰 반영 (2차)
PR #649 리뷰 지적 4건 처리:
1. **(차단) 컨트롤러 정상 경로 통합 테스트 누락** → `AdminAiExplanationControllerTest` 추가(REVIEWER 202·SUPER_ADMIN 202·admin_role 부족 403). `@WebMvcTest`+`@Import(AdminAiAuthentication)` 패턴.
2. **감사 조회 연동 미완** → AD-07 `AuditQueryService`는 `actionType IN(AI_*) AND targetType=AI_GENERATED_ASSET` 단일 타깃 필터라, target=`QT_PASSAGE`인 본 액션을 넣으려면 조회 쿼리 의미를 바꿔야 함(기존 AI 자산 뷰·테스트 영향). **본 PR 범위 밖으로 두고 후속 PR로 분리**(감사 조회 뷰를 본문/일반 대상까지 확장). 감사 로그 자체는 정상 기록됨(F-06 기록 요건 충족).
3. **트랜잭션 분리 정합성 리스크** → `generateQtPassageExplanation`의 `@Transactional` 제거. 시딩(NOT_SUPPORTED)으로 커밋된 job을 외부 tx가 감쌀 수 없어 "job 잔존·감사 롤백"의 잘못된 원자성 인상을 주던 것을 해소. 감사는 `AuditService#write` 자체 tx로 기록, 재시도는 활성 job 유니크로 멱등.
4. **`TODAY_PASSAGE_NOT_FOUND` 의미 오용** → seedForPassage 본문 부재 분기를 `PASSAGE_NOT_FOUND("QT_PASSAGE_NOT_FOUND")`로 정정(지정 본문 부재 ≠ 오늘 본문 부재).
