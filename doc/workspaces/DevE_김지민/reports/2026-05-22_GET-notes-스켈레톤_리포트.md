## 2. 오늘 학습한 핵심 개념 7개

### 2-1. record vs class — 불변(Immutable) DTO

DTO는 데이터 봉투라서 **값이 변경되면 안 됨**. record는 모든 필드가 자동 `final`이라 한 번 만들면 변경 불가. class는 setter로 값 변경 가능 → DTO 부적합.

```java
public record NoteListItem(Long id, String title, ...) {}   // 한 줄로 끝
// → 생성자, getter, equals, hashCode, toString 자동 생성
```

**한 줄 외움**: "record = 한 번 만들면 필드 값이 변하지 않는 데이터 컨테이너"

### 2-2. `long` vs `Long` — primitive vs Wrapper

| | `long` | `Long` |
| --- | --- | --- |
| 종류 | primitive (원시 타입) | wrapper class (객체) |
| null 가능 | ❌ | ✅ |
| 기본값 | `0L` | `null` |

**DTO에서**: PK(`id`)는 null 가능 → `Long`. 페이지 번호·개수는 항상 값 → `int`/`long` primitive.

### 2-3. JSON 직렬화 — `@RestController` 자동 변환

`@RestController`가 메서드 반환값(자바 객체)을 Jackson을 통해 JSON 문자열로 자동 변환.

비유: 요리(자바 객체) → 택배 박스(JSON)로 자동 포장 → 배달(HTTP 응답).

```java
return ApiResponse.success(response);  // ← 객체 그대로 반환
// → Spring이 자동으로 {"success":true,"data":{...}} JSON 문자열로 변환
```

### 2-4. `@PageableDefault(size = 20)` — 페이지 항목 수

`size` = **한 페이지에 담을 글 수** (페이지 수 아님). 명세 §1.6 공통 페이징 응답 형식.

| 글 125개 + size=20 | 결과 |
| --- | --- |
| `page=0` | 1~20번째 글 (첫 페이지) |
| `page=6` | 121~125번째 글 (마지막, 5개만) |
| `totalPages` | 7 (125÷20=6.25 → 올림) |

### 2-5. `?category=PRAYER` → `NoteCategory.PRAYER` 자동 변환

Spring이 자동으로 `NoteCategory.valueOf("PRAYER")` 동작. **대소문자 정확히 일치해야** 함. 매칭 실패 시 자동으로 `400 Bad Request`.

### 2-6. `implements` — 자바 강제 + IDE 시그니처 자동 + `throw`로 미구현 표시

| 누가 | 뭘 하나 |
| --- | --- |
| 자바 컴파일러 | "인터페이스의 모든 메서드 구현해야 한다"고 **강제** |
| IDE | `Alt+Enter` "Implement methods" → **시그니처만** 자동 입력 |
| 개발자 | **본문은 무조건 채워야 함** |

본문 채우기 옵션:
- `return null;` ❌ 호출자가 NullPointerException
- `throw new UnsupportedOperationException("TODO: ...");` ✅ 호출 즉시 명확한 메시지로 폭발

### 2-7. git staging area — `git add`로 파일별 commit 분리

```
Working Directory  →  Staging Area  →  Repository
   (실제 파일)        (git add)        (git commit)
```

`git add <파일>`로 특정 파일만 staging에 올림 → `git commit`은 staging에 있는 것만 묶어 기록. **파일을 추가 수정 안 해도 commit을 그룹별로 분리 가능**. 택배 박스에 짐 골라 담는 것과 같음.

**오늘 적용**: 워크플로우 doc 따로 → 코드 5개 따로 → commit 분리 (squash merge로 어차피 합쳐지지만 history 깔끔).
