### 1-3. `DevSecurityConfig.java` 신규 작성 (이번 작업의 유일한 코드 산출물)

- **위치**: `qtai-server/src/main/java/com/qtai/security/DevSecurityConfig.java`
- **목적**: 이승욱의 정식 JWT 인증 필터(`SecurityConfig`) 완성 전까지 dev 환경에서 인증 우회
- **핵심**: `@Profile("dev")` 로 운영 환경(prod)과 완전 분리

```java
@Configuration
@EnableWebSecurity
@Profile("dev")
public class DevSecurityConfig {

    @Bean
    SecurityFilterChain devFilterChain(HttpSecurity http) throws Exception {
        return http
                .csrf(csrf -> csrf.disable())
                .sessionManagement(sm -> sm.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
                .authorizeHttpRequests(auth -> auth.anyRequest().permitAll())
                .build();
    }
}
```

> IDE hint 발견 → `@Bean` 메서드의 `public` 제거 (Spring 6.x 권장).

## 2. 오늘 학습한 핵심 개념 5개

### 2-1. Entity 1개 vs Category 1개

**`notes` 테이블은 1개**. Entity 클래스도 `Note.java` 1개. 5종 노트(MEDITATION/SERMON/PRAYER/REPENTANCE/GRATITUDE)는 같은 테이블 안의 행으로 저장되고, **`category` 컬럼 값으로 구분**.

→ 김지민이 다루는 건 PRAYER/REPENTANCE/GRATITUDE 3종 데이터 (Entity 자체는 이지윤 소유).

### 2-2. `@Profile` 로 환경별 분리

운영 환경 코드를 건드리지 않고도 dev 환경 전용 설정을 추가할 수 있는 핵심 어노테이션.

```java
@Profile("dev")    // dev 환경에서만 활성화
@Profile("!dev")   // dev 아닌 환경에서 활성화
@Profile("prod")   // prod 환경에서만 활성화
```

운영(prod) 환경 코드는 이승욱 영역이라 손대지 않고, dev 전용 클래스만 추가해서 충돌 없이 개발 진행.

### 2-3. enum 생성자 특이점

- 생성자 이름은 **enum 클래스 이름과 동일** (예: `NoteCategory(String displayName) {...}`)
- 자동으로 `private` 처리 (외부에서 `new NoteCategory(...)` 호출 불가)
- enum 선언부 (`PRAYER("기도제목")`) 에서만 생성자 호출됨
- Lombok `@AllArgsConstructor`, `@RequiredArgsConstructor` 는 enum에 적용 안 됨 → 생성자 직접 작성
- Lombok `@Getter` 는 enum에 적용 가능 → getter는 자동 생성

### 2-4. Mock vs Dev 차이

| 구분 | Mock | Dev |
| --- | --- | --- |
| 대체 대상 | 진짜 객체(의존성) | 운영 환경 |
| 사용 패턴 | `@MockBean`, `XxxMock.java` | `@Profile("dev")`, `application-dev.yml` |
| 교체 방식 | 진짜 구현 들어오면 삭제 | 운영 환경에서 자동 비활성화 (코드는 남음) |

### 2-5. Spring 6.x `@Bean` 메서드 권장사항

```java
@Bean
public SecurityFilterChain ...  // ❌ public 권장 안 함 (이전 버전 스타일)

@Bean
SecurityFilterChain ...         // ✅ package-private (Spring 6.x 권장)
```

이유: `@Bean` 메서드는 Spring 컨테이너만 호출하므로 public 노출 불필요.

```powershell
git add qtai-server/src/main/java/com/qtai/security/DevSecurityConfig.java
```
