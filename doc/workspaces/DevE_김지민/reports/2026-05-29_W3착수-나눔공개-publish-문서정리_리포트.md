## 3. 나눔 공개(publish) 구현 — `POST /api/v1/notes/{id}/share` (F-10)

🟢 소크라테스 강 모드로 한 블록씩 진행. **5단계 중 ④까지 완료.**

| 블록 | 파일 | 내용 | 상태 |
|---|---|---|---|
| ① | `sharing/api/dto/PublishNoteRequest` | `@NotNull Boolean confirmNicknamePublic` + `Boolean commentsEnabled` | ✅ |
| ② | `sharing/internal/SharingPostRepository` | `existsByNoteId` (중복 공개 사전 차단) | ✅ |
| ③ | `sharing/internal/SharingPost` | 정적 팩토리 `publish(...)` — 스냅샷 박제·status=PUBLISHED·count=0 캡슐화 | ✅ |
| ④ | `sharing/api/PublishNoteUseCase` + `SharingPostService.publish()` + `ErrorCode.SHARING_POST_ALREADY_SHARED(S0003)` | 검증→노트조회→닉네임박제→스냅샷저장 (@Transactional 원자성), `toDetail` 매퍼 추출 | ✅ |
| ⑤ | `note/web/NoteController` `@PostMapping("/{noteId}/share")` | sharing 포트 호출 배선 | ⬜ **이월** |
| 테스트 | 단위·컨트롤러·통합 3종 + dev-console | | ⬜ **이월** |

**설계 결정**:
- 엔드포인트는 NoteController(경로가 /notes)가 sharing `PublishNoteUseCase`(api 포트) 호출 — ArchUnit 경계 OK(web→api, 순환 미검사 확인).
- 중복 공개: **사전 조회(existsByNoteId)→409**, DB UNIQUE가 최종 backstop.
- snapshotBody: body 우선, 비면(묵상 노트) 4섹션 라벨 합성.

## 4. 오늘 배운 개념

| 개념 | 한 줄 |
|---|---|
| 원시 vs 래퍼 | `boolean`은 null 불가(미전송도 false로 채움), `Boolean`은 null 가능 → `@NotNull`로 "안 보냄" 검출 |
| 정적 팩토리 vs 생성자 | 둘 다 객체 생성. 팩토리는 **이름**을 줄 수 있어 의도(publish) 표현 + 같은 파라미터로 여러 개 가능. (박제와는 무관 — 박제는 값 복사) |
| `@Transactional` readOnly 오버라이드 | 클래스가 readOnly여도 쓰기 메서드에 `@Transactional` 따로 붙여 INSERT 커밋 |
| 스냅샷(박제) | 공개 시점 값을 별도 컬럼에 복사 + 이후 안 건드림 → 원본 변경에 무영향 |
| ArchUnit 경계 | internal 접근·web→web만 막음, **모듈 순환은 미검사**. 도메인 간 api 호출은 허용 |
| 중복 차단 패턴 | 사전 조회(친절한 에러) vs DB 제약 catch(동시성 안전). 보통 둘 병행, 우린 사전조회+DB backstop |
