### ② 좋아요 `POST·DELETE /api/v1/sharing-posts/{id}/like`
- **설계 결정**: likeCount = **COUNT 재계산**(엔티티 증감 대신 — 정확성↑, MVP 규모에 적합) · 중복 **409 DUPLICATE_LIKE**(S0004 신규) · 취소 **멱등 204**.
- `ToggleLikeUseCase`+`LikeResponse`, `SharingPostService.like/unlike`, `PostLike.of`·`SharingPost.syncLikeCount`, `PostLikeRepository` 보강(existsBy·countBy·deleteBy), `SharingPostController` 2엔드포인트(**지민 직접**).
- 테스트 11개(서비스5·컨트롤러4·통합2). 부팅 검증: 201→409(중복)→204→204(멱등).

### ③ 댓글 `POST·GET /api/v1/sharing-posts/{id}/comments` + `DELETE /api/v1/comments/{id}`
- **설계 결정**: **평면 댓글**(대댓글 X — 04 명세에 parentId 없음, `parent_id`는 v2용) · 본문 **`body`**(04 일치) · commentCount **COUNT 재계산** · 삭제 **soft delete·본인만 403·멱등** · 댓글OFF **409 INVALID_STATUS_TRANSITION** · 닉네임 **실시간 조회**(getMemberPublic, 박제 아님).
- `CommentRepository` 신규, DTO 3개, `CommentUseCase`+**`CommentService`**(분리), `CommentController`(`/api/v1`, **지민 직접**).
- 테스트 16개(서비스8·컨트롤러6·통합2) + dev-console "댓글" 섹션.

## 2. 오늘 배운 개념

| 개념 | 한 줄 |
|---|---|
| COUNT 재계산 vs 엔티티 증감 | 좋아요·댓글 수는 실제 행을 세서 맞추면 항상 정확(++/-- 동시성 어긋남 회피). 작은 규모엔 쿼리 비용 무의미 |
| 멱등성(idempotent) | 같은 요청(취소·삭제)이 두 번 와도 안전하게 같은 상태로 수렴 → `deleteBy`(없으면 0건)·`if(!isDeleted)` |
| 파생 쿼리(derived query) | 메서드 **이름**이 곧 SQL — `findBy…IsDeletedFalseOrderByCreatedAtAsc`, `existsBy…`, `countBy…` |
| 정적 팩토리 `of()` | 생성자(protected) 대신 "~로 만든다"는 이름의 메서드(`Comment.of`, `PostLike.of`) — 자바 관례(List.of 등) |
| 박제 vs 실시간 조회 | 공유글 닉네임=박제(스냅샷 정책), 댓글 닉네임=실시간(getMemberPublic) — 댓글은 스냅샷 대상 아님 |
| 이중 방어(defense in depth) | 댓글OFF는 UI 차단 + **서버 409** 둘 다 — "클라이언트를 믿지 마라" |
| HTTP 상태코드 의미 | 201(생성)·204(본문없음)·400(검증)·403(권한)·404(없음)·409(충돌)는 클라가 흐름 제어에 쓰는 신호 |
| 수동 E2E/스모크 테스트 | dev-console로 진짜 앱·DB 끝까지 눌러보기 ↔ 자동 테스트(JUnit, 피라미드 아래가 두꺼움) |

## 3. 환경 메모 (중요)

- **로컬 `gradlew test` 실행 불가**: 경로에 한글(`프로젝트 - 2`) + Gradle9 테스트 워커 argfile 인코딩 문제로 모든 테스트 클래스 `ClassNotFoundException`. → **테스트 실행은 CI(Linux)**, 로컬은 `bootJar`/`compileTestJava`로 컴파일만 확인.
- **dev 부팅은 `java -jar`로만**: `bootRun`도 같은 argfile 문제. jar는 내부 클래스패스라 안전. JWT 키는 `src/test/resources/application.yml`의 테스트 키를 env로(JWT_PRIVATE_KEY/JWT_PUBLIC_KEY).
- `DevSecurityConfig`에 frameOptions sameOrigin 1줄 추가(H2 콘솔 흰화면 방지, dev 전용 — 커밋 대상).
- `dev-console.html`은 개인 dev 도구라 **`.gitignore` 처리**(PR 제외).
