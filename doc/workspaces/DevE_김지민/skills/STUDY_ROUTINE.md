# 김지민 공부 루틴 스킬

## 이 스킬의 목적
하루 6시간 (평일 기준, 휴일 제외), Spring Boot + Flutter 입문 수준에서
"일단 돌아가게 만들고 나중에 채운다" 원칙으로 진행하는 학습 루틴이다.
Claude Code는 이 스킬을 읽고 아래 5단계 루틴을 그대로 진행한다.

---

## 전제 원칙

- 전체를 다 이해하고 시작하려 하지 마. 지금 당장 필요한 것만 샘플링한다.
- 백엔드 없이 Flutter 화면 먼저 만들지 마. 연결할 때 다 뜯어고쳐야 한다.
- 에러 나면 30분 안에 해결 안 되면 바로 질문한다.
- SKILL.md 규칙(소크라테스식 대화)을 항상 함께 적용한다.

---

## 핵심 전략: 3-1-1 샘플링

구현 시작 전 항상 이 세 가지만 먼저 확인한다.

| 단계 | 내용 | 시간 |
|------|------|------|
| 개념 3개 | 이 기능에서 꼭 알아야 할 개념 3가지 | 15분 |
| 흐름 1개 | Controller → Service → Repository 전체 흐름 한 번 | 15분 |
| 질문 1개 | "내가 이해한 게 맞아?" 한 줄 확인 | 10분 |

---

## 하루 5단계 루틴 (6시간)

### ① 시작 전 — 오늘 뭐 할지 샘플링 (40분)

Claude Code가 해야 할 것:
1. 워크플로우 파일에서 오늘 구현할 항목 1~2개를 함께 확인한다.
2. 3-1-1 샘플링을 진행한다.
   - 오늘 기능의 핵심 개념 3개를 물어본다. (코드 없이 말로만)
   - Controller → Service → Repository 흐름을 한 줄씩 설명한다.
   - "내가 이해한 게 이거야 — [요약]. 맞아?" 라고 물어보고 확인한다.
3. 개념이 이해되면 구현 시작. 안 되면 추가 질문 먼저.

```
[사용 예시]
STUDY_ROUTINE.md 읽고 루틴 시작해줘.
오늘 [기능명] 구현할 거야.
```

---

### ② 구현 1회차 — 첫 번째 기능 완성 (2시간)

Claude Code가 해야 할 것:
1. SKILL.md 소크라테스 규칙을 함께 적용한다.
   - 구현 방법 선택지 제시 → 내가 고르기 → 이유 설명
   - 코드 블록마다 "왜 이렇게 짰냐면:" 설명 추가
   - 틀린 이해는 정답 대신 질문으로 유도
2. 구현 순서는 반드시 Entity → Repository → Service → Controller 순서.
3. 각 클래스 작성 후 "이 클래스가 하는 일을 한 줄로 말해볼 수 있어?" 질문.

막힐 때 쓸 질문 템플릿:
```
이 코드에서 [모르는 부분]이 왜 필요한 거야?
[A 방식]이랑 [B 방식]의 차이가 뭐야?
```

> Claude Code가 짠 코드 절대 그냥 복붙 금지.
> 한 블록 끝날 때마다 "왜 이렇게 짰어?" 한 번씩 꼭 물어봐.

---

### ③ 중간 복습 — 1회차 설명 복습 (30분)

구현 1회차가 끝나면 Cowork에서 진행한다. (Claude Code 아님)

Cowork에서 이렇게 말해:
```
오늘 [기능명] 만들었어. 내가 이해한 거 설명할게.
[설명]
맞아?
```

Claude Code가 해야 할 것:
1. 구현 1회차가 끝나면 "이제 Cowork에서 설명 복습하고 와." 라고 안내한다.
2. 복습 후 돌아오면 2회차 시작.

---

### ④ 구현 2회차 — 두 번째 기능 완성 (2시간)

2회차도 ② 와 동일한 방식으로 진행.
하루 목표: 기능 2개 완성.

---

### ⑤ 마무리 — 내일 준비 (50분)

Claude Code가 해야 할 것:
1. 오늘 완성한 기능 2개를 체크리스트에서 `[ ]` → `[x]` 로 업데이트한다.
2. "오늘 배운 핵심 개념을 각각 한 줄로 정리해줄 수 있어?" 질문한다.
3. 워크플로우에서 내일 할 항목을 확인하고 한 줄로 요약해준다.
