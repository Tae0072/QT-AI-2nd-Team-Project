> 앞선 노트(N-04 family·invalidate 등)와 겹치는 건 생략, M-05에서 새로 쓴 것 위주.

---

## 1. PopupMenuButton — ⋮ 오버플로우 메뉴

```dart
PopupMenuButton<String>(
  onSelected: (action) => _onAction(context, ref, action),  // 고른 value가 들어옴
  itemBuilder: (_) => [
    if (post.isHidden)
      const PopupMenuItem(value: 'show', child: Text('공개로 되돌리기'))
    else
      const PopupMenuItem(value: 'hide', child: Text('숨기기')),
    const PopupMenuItem(value: 'delete', child: Text('삭제')),
  ],
)
```
- `<String>` = 각 항목의 `value` 타입. 누르면 `onSelected(value)` 호출.
- **`itemBuilder` 안에서 `if/else`로 메뉴를 동적 구성** 가능 → 상태(`isHidden`)에 따라 다른 메뉴.
- 핵심: **공개중인 글에 '되돌리기'가 뜨면 안 되므로** `isHidden`으로 분기.

## 2. 상태로 UI 분기 — 모델에 의도 헬퍼

```dart
bool get isHidden => status == 'HIDDEN';   // 모델에 둠
```
- 화면 곳곳에서 `status == 'HIDDEN'` 문자열 비교를 반복하지 않고, **모델 getter 하나로** → 오타·중복 방지, 의도가 드러남. (N-04의 `isFreeNote`와 같은 패턴)

## 3. 액션 → 변경 → invalidate (두 곳)

```dart
await repo.hidePost(post.id);          // 또는 showPost / deletePost
ref.invalidate(mySharingPostsProvider); // 내 목록 새로고침
ref.invalidate(sharingPostsProvider);   // 공개 피드도(숨김/삭제는 피드에 영향)
```
- 변경 후 **관련 provider를 모두 무효화**해야 화면이 최신으로 다시 그려진다.
- 왜 둘 다? **내 목록**뿐 아니라 **공개 피드**에서도 그 글이 사라지/나타나므로.

## 4. 삭제 = 확인창 (되돌리기 어려운 동작)

```dart
final ok = await showDialog<bool>(context: context, builder: (ctx) => AlertDialog(
  actions: [ ...취소(pop false)..., ...삭제(pop true)... ],
));
if (ok != true) return;   // 취소/닫기면 중단
```
- 08 §8.2: 삭제·공유·신고처럼 **되돌리기 어려운 동작은 확인 절차**. 숨김/되돌리기는 가역이라 확인 없이 바로.

## 5. 진입점 — AppBar actions

```dart
appBar: AppBar(title: const Text('나눔'), actions: [
  IconButton(icon: Icon(Icons.manage_accounts_outlined),
    onPressed: () => Navigator.pushNamed(context, AppRouter.mySharing)),
]),
```
- AppBar 우측 `actions: [IconButton...]` 로 화면 진입점을 둔다. (06은 마이페이지 진입이나 팀 합의로 피드 진입 → 문서 반영 요청)

## 6. 재사용한 기존 패턴 (새로 안 만듦)
- `whenOrDefault`(로딩/에러/빈화면), `EmptyView`, `RefreshIndicator`(당겨 새로고침), 엔벨로프 `['data']` 언랩, 페이징 `content`+`last→hasNext`, family 아닌 단일 `FutureProvider.autoDispose`.

---

## 한 줄 정리
- 상태 따라 메뉴가 갈리면 **`PopupMenuButton` + 모델 `isHidden` getter + itemBuilder if/else.**
- 변경 후엔 **영향받는 provider 전부 invalidate** (내 목록 + 공개 피드).
- 되돌리기 어려운 삭제만 확인창, 가역 동작(숨김/되돌리기)은 즉시.
