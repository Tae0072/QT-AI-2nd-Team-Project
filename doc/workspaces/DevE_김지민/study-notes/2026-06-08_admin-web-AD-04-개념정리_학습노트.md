## 0. 큰 그림 — AD-07에서 뭐가 늘었나

- **목록 부분은 그대로 재사용**: `useList(listReports)` + `Table` + 페이징. (AD-07에서 도구로 빼둔 보람 — 새 화면이 빨라짐)
- **새로 추가된 것 = "액션"**: 신고를 처리/반려하는 동작. 핵심은 ① 모달 ② 처리 후 새로고침.

## 1. antd `Modal` — 가운데 팝업으로 액션 받기

```tsx
const [actionTarget, setActionTarget] = useState<ActionTarget>(null);
...
<Modal
  open={actionTarget != null}      // 상태가 있으면 열림
  onOk={onSubmitAction}            // [확인] → API 호출
  onCancel={() => setActionTarget(null)}  // [취소] → 닫기
  confirmLoading={submitting}      // 제출 중 버튼 로딩
  destroyOnClose                   // 닫을 때 내부 폼 초기화
>
  ...폼...
</Modal>
```
- **모달은 "열림/닫힘"을 상태(state)로 제어**한다. `open`이 true면 보이고 false면 숨김.
- 여기선 `actionTarget`(어느 신고 + 처리/반려) 하나로 열림 여부까지 같이 표현 → `!= null`이면 열림.

## 2. 처리·반려를 **공용 모달 하나**로

```tsx
type ActionTarget = { report: Report; type: 'resolve' | 'reject' } | null;
const isResolve = actionTarget?.type === 'resolve';
```
- 처리(resolve)·반려(reject)는 입력만 조금 다르고 흐름은 같다 → **모달을 두 개 만들지 않고 하나로**.
- `type`으로 분기: 제목·버튼 텍스트, 그리고 `isResolve`일 때만 "조치·알림" 입력칸을 보여줌.
- **핵심: 비슷한 두 동작은 상태(type)로 구분해 한 컴포넌트로 묶는다.** (중복 줄이기)

## 3. 모달 안 폼 — `Form.useForm()` + `validateFields()`

```tsx
const [modalForm] = Form.useForm();
...
let values;
try { values = await modalForm.validateFields(); } // 필수값 검증
catch { return; }                                   // 검증 실패면 모달 유지
```
- `Form.useForm()` = 폼을 코드에서 다루는 손잡이. `validateFields()`로 한 번에 검증.
- 검증 실패하면 throw → `catch { return }`로 **모달 안 닫고** 사용자가 고치게 둔다.
- `resetFields()`를 모달 열 때마다 호출 → 이전 입력 안 남게.

## 4. 액션 게이팅 — "접수(RECEIVED)일 때만" 버튼

```tsx
render: (_, r) => r.status === 'RECEIVED'
  ? <처리/반려 버튼들 />
  : <span>—</span>
```
- 상태는 **접수 → 처리됨 / 반려** 3개("처리중"은 없음). [처리] 누르면 바로 처리됨으로.
- **이미 처리/반려된 신고엔 버튼을 숨김** → 중복 처리 방지. (이미 결정난 걸 또 처리할 이유 없음)

## 5. 처리 후 `reload()` — 화면을 서버와 맞추기

```tsx
await resolveReport(id, payload);   // 서버 상태: 접수 → 처리됨
reload();                           // 목록 다시 받아 화면에 반영
```
- 처리하면 **서버**의 상태는 바뀌지만, 화면 표는 아직 **옛 데이터(접수)** 를 들고 있음.
- `reload()`(= useList가 준 함수)로 **다시 받아와야** 표에 "처리됨"이 보인다. 안 부르면 처리했는데도 접수로 보임.

## 6. 곁들여 익힌 antd 소품

- **`Select`**(필터): 고정 보기(상태·대상)는 입력칸보다 드롭다운이 깔끔. `options`로 목록 전달.
- **`Tag`**(상태): 상태 코드를 색 배지로(`접수`=gold, `처리됨`=green). `STATUS_META` 맵으로 코드→라벨·색.
- **컬럼 안 클로저**: 액션 버튼이 `openAction`을 써야 해서 columns를 **컴포넌트 안에** 정의(밖 상수면 함수 접근 불가).
