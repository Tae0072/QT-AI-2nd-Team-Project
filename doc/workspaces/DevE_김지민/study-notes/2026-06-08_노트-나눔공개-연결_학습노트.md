## 2. 바텀시트가 "취소 vs 확정"을 어떻게 구분하나 — `bool?` 반환

```dart
Future<bool?> showNotePublishSheet(BuildContext context) {
  return showModalBottomSheet<bool>( ... );  // 댓글허용 bool 반환
}
// 호출부
final commentsEnabled = await showNotePublishSheet(context);
if (commentsEnabled == null || !context.mounted) return; // null = 취소/닫기
// 여기 오면 확정 → commentsEnabled(true/false) 사용
```
- `Navigator.pop()` → **null** (취소), `Navigator.pop(true/false)` → 확정 + 댓글허용 값.
- `bool?`(nullable) 하나로 **3가지 상태**(취소 / 켜고확정 / 끄고확정)를 표현. null 체크가 곧 "취소했나?" 판정.

## 4. 평범한 Text를 탭 가능하게 — `InkWell`


- 스낵바(그 순간) + 뱃지(언제든) = 역할이 달라 **둘 다** 둠.

## 5. 다른 feature의 provider 가져다 쓰기 (크로스 피처 import)


- Flutter는 **feature끼리 import 허용**(서버의 도메인 경계 규칙과 다름).

# 디버깅하며 배운 것 (웹 실행 삽질 모음)

## 7. CORS — 왜 웹에서만 막히나

- **증상**: `Access to XMLHttpRequest ... blocked by CORS policy: No 'Access-Control-Allow-Origin'`
- **원인**: 브라우저 보안 규칙. `A사이트(:5500)`의 JS가 `B서버(:8080)`를 부르면, 서버가 "A를 허용"이라고 **응답 헤더로 명시**해야 통과.
- 서버 dev CORS 허용목록엔 **`localhost:3000`만** 있었음([DevSecurityConfig.java](../../../../qtai-server/src/main/java/com/qtai/security/DevSecurityConfig.java)) → 5500은 차단.
- **해결**: 웹앱을 **3000 포트**로 실행 (`--web-port=3000`). 서버 코드 안 건드림.
- **핵심**: CORS는 **브라우저(웹)에서만** 발생. 에뮬레이터·실폰은 CORS 없음.

## 8. 웹 dev 인증 — `X-Dev-User-Id` 헤더 (3박자 플래그)

- 웹은 카카오 로그인 미지원 → dev 우회 사용.
- 서버는 `X-Dev-User-Id` 헤더로 "누구인지" 판단([DevUserIdHeaderFilter](../../../../qtai-server/src/main/java/com/qtai/security/DevUserIdHeaderFilter.java) — **DB 조회 안 하고 헤더값 그대로 신뢰**).
- 그 헤더는 **3박자**가 다 맞아야 붙음: `kIsWeb` + `--dart-define=WEB_DEV_NO_LOGIN=true` + dev빌드.
- 플래그 빠지면 → 헤더 없음 → 쓰기(POST) **401**. 또는 로그인 화면으로 튕김(`DEV_FORCE_HOME`도 빠진 경우).
- **실행 명령(웹, 한 줄)**:
  ```
  flutter run -d chrome --web-port=3000 --dart-define=DEV_FORCE_HOME=true --dart-define=WEB_DEV_NO_LOGIN=true --dart-define=WEB_DEV_USER_ID=1
  ```
  - `DEV_FORCE_HOME` = 로그인 건너뛰고 홈 / `WEB_DEV_NO_LOGIN` = 인증헤더 부착 / `WEB_DEV_USER_ID=1` = 그 회원으로.

## 9. 로컬 DB 시드 — "목데이터 없음"

- `X-Dev-User-Id=1`은 헤더만으로 인증은 통과하지만, **노트 저장·나눔 공개는 진짜 회원 1번이 DB에 있어야** 함(노트가 member에 묶이고, 공유 시 닉네임 박제).
- 해결: 로컬 MySQL에 회원 1줄.
  ```sql
  INSERT INTO members (id, kakao_id, nickname) VALUES (1, 1000001, 'devuser');
  ```
- **다른 팀원 영향 없음** — MySQL은 **각자 PC의 로컬 docker 컨테이너**. 공용 원격 DB 아님. git에도 안 올라감(코드 아니라 런타임 데이터).
