## AD-07 · 감사 로그 `/audit-logs` ✅ (담당: 팀원)

- **무슨 화면**: 관리자/시스템(SYSTEM_BATCH)의 주요 행위(QT 발행·AI 승인·신고 처리 등) 기록을 조회·필터.
- **연결 API**: `GET /api/v1/admin/audit-logs`
  - 요청(쿼리): `actorType`(ADMIN·SYSTEM_BATCH), `actionType`(예: AI_ASSET_APPROVE), `from`·`to`(YYYY-MM-DD), `page`·`size`
  - 응답(`Page<AuditLog>`): `content[]`(id·actorType·actorLabel·actionType·targetType·targetId·createdAt …) + 페이징
- **주요 기능**: 필터(주체·행위·기간) + 표(시각·주체·행위·대상) + 페이지네이션. **읽기 전용**(수정/삭제 없음).
- **프런트 stub**: `api/auditLogs.ts` `listAuditLogs(params)`.

## AD-04 · 신고 처리 `/reports` ✅ ← 김지민 다음 작업

- **무슨 화면**: 신고된 글/댓글/AI응답을 검토해 숨김·제재·반려 처리.
- **연결 API**:
  - `GET /api/v1/admin/reports` — 필터 `status`(RECEIVED·RESOLVED·REJECTED)·`targetType`(POST·COMMENT·AI_QA_REQUEST·AI_ASSET) + 페이징
  - `POST /api/v1/admin/reports/{id}/resolve` — 바디 `{action: HIDE_TARGET|SUSPEND_USER, reason, notifyReporter}`
  - `POST /api/v1/admin/reports/{id}/reject`
- **주요 기능**: 목록+필터+페이징 + **처리 모달**(action 선택·사유·신고자 알림 토글). 처리 후 목록 갱신.
- **프런트 stub**: `api/reports.ts` `listReports·resolveReport·rejectReport`.

## AD-03 · AI 산출물 검증 `/ai-assets` ✅ (담당: 팀원)

- **무슨 화면**: 사전 생성된 해설/시뮬레이터 산출물을 승인/반려/숨김/재생성.
- **연결 API**:
  - `GET /api/v1/admin/ai/assets` — 필터 `assetType`·`targetType`·`status`(VALIDATING·APPROVED·REJECTED·HIDDEN) + 페이징
  - `GET .../{id}` 상세 · `POST .../{id}/approve|reject|hide|regenerate`
- **주요 기능**: 목록+필터 + 승인/반려/숨김 모달 + 재생성(202). **⚠️ 승인 전 원문(검증 주석) 비노출**(금지규칙). 출처 표기 여부 표시.
- **프런트 stub**: `api/aiAssets.ts` `listAiAssets·getAiAsset·approve·reject·regenerate`.

## AD-08 · AI 운영 모니터링 `/ai-monitoring` ✅

- **무슨 화면**: 기간 내 AI 생성잡·검증·Q&A·체크리스트 통과율을 집계 카드로.
- **연결 API**: `GET /api/v1/admin/ai/monitoring?from=&to=`
  - 응답: `generationJobs{queued,running,succeeded,failed}`, `validation{passCount,failCount,failureReasons[]}`, `qa{requested,answered,blocked,…}`, `checklists[]`.
- **주요 기능**: 기간 필터 + 집계 **카드/통계 표시**(목록 아님 → useList 대신 단건 조회 훅 또는 직접 호출).
- **프런트 stub**: `api/aiMonitoring.ts` `getAiMonitoring`.

## 공통 패턴 (목록형 화면 = AD-07/04/03/05)

1. `useList(목록API, {page:0, size:20})` 로 데이터·로딩·필터 상태 관리(에러는 토스트).
2. 필터 폼 → `setParams({...필터, page:0})`.
3. antd `Table`(`rowKey`, columns) + `pagination`(0-based↔1-based 변환, total=`totalElements`).
4. 액션(승인/반려/처리)은 모달 → API 호출 → `reload()`.
> AD-08은 목록이 아니라 **집계 단건**이라 패턴이 다름(카드 표시).
