역할 핵심 선언
김지민 — Flutter 화면, 관리자 화면, Today QT 사용자 흐름
1 첫 화면은 Today QT 중심이며 별도 마케팅 홈을 만들지 않는다.
2 미인증 기능 화면 차단과 로그인 후 접근 기준을 명확히 한다.
3 시뮬레이터 버튼은 READY일 때만 활성화하고 MISSING/FAILED/DISABLED는 비활성화한다.
4 AI 자유 챗봇, AI 찬양 추천, 교회 인증 버튼은 MVP 화면에 넣지 않는다.
본인 우선 액션 4가지
1 Today QT 화면에서 본문·해설·묵상·시뮬레이터 상태를 모두 표시
2 미인증 기능 화면 차단과 로그인 후 접근 기준을 구분
3 관리자 메뉴 범위를 오늘 QT, 승인 해설, 시뮬레이터, 신고, 찬양 큐레이션으로 제한
4 금지 UI 버튼이 화면에 남지 않았는지 PR 전 확인
중심 영역
mobile / admin
작업 요약
Today QT 화면, 로그인 경계, 묵상 작성, 마이페이지, 묵상 달력, 관리자 운영 화면
담당 경로 (구현 저장소 기준)
  • flutter-app/lib/features/today_qt/ — Today QT 화면
  • flutter-app/lib/features/note/ — 묵상 작성·달력
  • flutter-app/lib/features/admin/ — 관리자 화면
  • flutter-app/test/ — 화면/상태 테스트
  • workspaces/김지민/ — workflow/report 기록

소유권 선언 (Code Ownership)
실제 파일 생성과 코드는 별도 구현 GitHub에서 진행한다. 이 문서 저장소에는 구현 골격을 만들지 않는다.
flutter-app/lib/
Flutter 사용자 화면
OWNER
flutter-app/lib/admin/
관리자 화면
OWNER
flutter-app/test/
화면 테스트
OWNER
workspaces/김지민/
개인 워크플로우와 리포트
OWNER
2
PR 가이드 — 이 섹션만 읽으면 작업 흐름 확인 가능
다운로드본 Lead 실행가이드의 패널형 양식과 체크리스트 밀도를 반영했다. 단, Kafka/K8s/SSE/세션형 AI 같은 구버전 내용은 v3.1 금지 기준으로 바꿨다.
