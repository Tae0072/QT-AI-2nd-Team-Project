## 2. 고정 기준

| 구분 | 기준 |
| --- | --- |
| 첫 진입 | 로그인 전에는 로그인 화면으로 진입하고, 로그인 후 별도 홈 화면 없이 Today QT 화면으로 이동한다. |
| 로그인 경계 | Today QT, 성경 본문, 저장, 해설, 찬양, 달력 등 기능 화면은 로그인 후 접근 |
| 닉네임 설정 | 카카오 로그인 후 닉네임 없으면 A-03으로 이동. 카카오 닉네임을 기본값으로 자동 채움. 가입 첫 설정은 7일 잠금 면제. |
| 닉네임 변경 (M-04) | 마지막 변경 후 7일 미경과 시 입력 잠금 + 잠금 해제 시각(`nicknameUnlockAt`) 안내. `409 NICKNAME_CHANGE_LOCKED` 응답 처리. |
| Today QT 100% | 시뮬레이터 클립이 없어도 상태값을 표시하고 버튼을 비활성화하면 정상 흐름이다. |
| 시뮬레이터 버튼 | `READY`만 활성화, `MISSING`/`FAILED`/`DISABLED`는 비활성화 |
| `@`멘션 인용 | 노트 본문 작성 중 `@` 트리거 → 책 자동완성 → `GET /api/v1/bible/verses?bookCode=&chapter=&verseFrom=&verseTo=` 호출 → 인용 블록 삽입 + `verseIds` 배열 동기화. F-03 QT 노트와 F-16 자유 노트 모든 카테고리에서 사용 가능. |
| 한글 성경 로컬 | 앱 첫 실행 시 `GET /api/v1/bible/bundle?language=ko`로 한글 번들 다운로드 → 로컬 SQLite 적재. 이후 한글 본문 조회는 로컬에서. 영어 본문은 온라인 조회만(`GET /api/v1/bible/verses`). 오프라인에서 한글 본문은 표시, 영어 본문은 "오프라인" 상태 표시. |
| 관리자 UI | **사용자 Flutter 앱에 관리자 화면(AD-) 포함 금지**. 별도 웹 프런트엔드(`admin-web/`)에서 구현. SSR/SPA 채택은 v1.2 이후 결정. |
| 금지 UI | AI 자유 챗봇, AI 찬양 추천, 교회 인증 버튼은 MVP 화면에 두지 않는다. |
| 관리자 | 오늘 QT, 승인 해설, 시뮬레이터 상태, 신고, 찬양 큐레이션을 관리 대상으로 둔다 (관리자 웹에서). |

## 5. 브랜치·PR·워크플로우 기록 규칙

| 항목 | 김지민 기준 |
| --- | --- |
| 브랜치 생성 | `dev` 최신화 후 `feature/mobile-*`, `feature/admin-*`, `bugfix/mobile-*` 형식으로 생성 |
| PR 대상 | 항상 `dev` |
| 자동 PR | 브랜치 push 후 GitHub PR을 생성하면 CI와 자동 리뷰가 실행된다. 자동 머지는 하지 않는다. |
| 작업 폴더 | 별도 구현 GitHub에서 `workspaces/김지민/workflows`, `workspaces/김지민/reports`를 사용 |
| 기록 순서 | 작업 전 workflow 작성 → 작업·검증 → report 작성 → PR 생성 |
| 기준 문서 | 상세 명령과 템플릿은 `개발자별_일정표/00_공통_브랜치_PR_워크플로우_규칙.md`를 따른다. |
